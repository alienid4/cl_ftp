# Runbook: Linux 用戶 PoC 部署 (C:\ only, no HTTPS)

| 項目 | 值 |
|---|---|
| **版本** | v1.0.0.10 |
| **日期時間** | 2026-05-20 08:30 |
| **適用對象** | Linux 背景, 不熟 Windows; 只能用 C:\ |
| **預期結果** | SF 主機可從 LAN 訪問 `http://<SF-IP>:5000/` Portal |
| **預估耗時** | 5-10 分鐘 (有 wheels 的話) / 之後另外 5 分鐘抓 wheels |

---

## 前提條件

- [ ] SF 主機 Windows Server 2022 已部署 sf_offline_bundle (放桌面或 C:\install\)
- [ ] 已套 v1.0.0.7 + v1.0.0.8 + v1.0.0.9 patch (這次補套 v1.0.0.10)
- [ ] PowerShell 用**系統管理員**模式打開
- [ ] SF 主機本機能上 GitHub (raw.githubusercontent.com)

如果 SF 主機完全沒外網, 改在外網 PC 抓檔, 拷 USB 帶進去 (見[附錄 A](#附錄-a-無外網替代方案))。

## 操作步驟 (照順序貼進 PowerShell)

### Step 1: 套 v1.0.0.10 patch (3 秒)

```powershell
# 抓 + 解壓 + 套用
iwr https://github.com/alienid4/cl_ftp/raw/main/release-zip/sf-patch-v1.0.0.10.zip -OutFile p10.zip
Expand-Archive p10.zip -DestinationPath p10 -Force
cd p10
.\run_patch.cmd
# 看到「套用 patch 到此 SF bundle? (Y/n)」按 Y
```

**這個 patch 修了** (Linux 對照):

| 修了什麼 | Linux 對照 |
|---|---|
| sshd 啟動失敗 (DenyUsers SF\Administrator) | `vi /etc/ssh/sshd_config` 移掉錯的 DenyUsers |
| 04 帳號 prompt 卡住 | 加 `--batch` flag 像 `useradd --no-interactive` |
| 09 pip 連 pypi.org timeout | 改 `pip install --index-url=file://...` 純離線 |

### Step 2: 一鍵 C:\ 部署 (3 分鐘)

```powershell
# 回家目錄, 抓並跑 poc 腳本
cd $env:USERPROFILE
iwr https://raw.githubusercontent.com/alienid4/cl_ftp/main/scripts/poc_setup_c_drive.ps1 -OutFile poc.ps1
.\poc.ps1
```

腳本內部會做 8 步, 每步附 Linux 對照註解:

| Step | 動作 | Linux 對照 |
|---|---|---|
| A | 改 SQL schema 路徑 D:→C: | `sed -i 's|D:\\\\_portal|C:\\\\_portal|' schema.sql` |
| B | 建目錄 | `mkdir -p /data/exchange /portal` |
| C | 設 NTFS ACL | `chown/chmod` |
| D | 重建 SQL DB | `mysql < schema.sql` |
| E | 重啟 sshd | `systemctl restart sshd` |
| F | 開防火牆 5000 | `firewall-cmd --add-port=5000/tcp` |
| G | 部署 Portal | `pip install -r requirements.txt` |
| H | 註冊 Portal service | `systemctl enable portal` |

腳本最後**直接顯示**:
- 服務狀態 (對應 `systemctl status`)
- Port 監聽 (對應 `ss -tlnp`)
- IP 位址 (對應 `ip a`)
- **訪問網址** (`http://<SF-IP>:5000/`)

### Step 3: 跑診斷 (10 秒)

```powershell
iwr https://raw.githubusercontent.com/alienid4/cl_ftp/main/scripts/diagnose_portal.ps1 -OutFile diag.ps1
.\diag.ps1
```

5 個 check, 告訴你 Portal 訪問空白的原因 (5 種可能)。

## 預期結果

### 沒抓 wheels 時 (常見起手)

Step 2 結尾:
```
[Step G] 部署 Portal
  [warn] C:\install\python_wheels 沒有 wheels, 跳過 Portal 部署

服務狀態:
  sshd                Running  Automatic  ✓
  MSSQL$SQLEXPRESS    Running  Automatic  ✓
  W3SVC               Running  Automatic  ✓
  FileExchangePortal  (沒有, 因為沒裝)
```

訪問 `http://<SF-IP>:5000/` **仍空白**, 這是正常的, 還缺 Step 4 抓 wheels。

### Step 4: 補抓 Python wheels (給 Portal 跑起來)

**外網 PC** (有 Internet 的 Win 機器):
```powershell
mkdir C:\Temp\sf_wheels
cd C:\Temp\sf_wheels
pip download flask waitress pyodbc python-ldap requests
# 大約 16 MB, 36 個 .whl 檔
```

USB 拷整個 `sf_wheels\` 到 SF 主機 `C:\install\python_wheels\`:
```powershell
# SF 主機
mkdir C:\install\python_wheels -Force
# USB E: 拷貝
Copy-Item E:\sf_wheels\* C:\install\python_wheels\ -Recurse -Force
```

**重跑 poc.ps1**:
```powershell
cd $env:USERPROFILE
.\poc.ps1
```

這次 Step G 會找到 wheels, 跑離線安裝, Portal 起來。

## 訪問 Portal

```
http://<SF-IP>:5000/                   ← 主入口
http://<SF-IP>:5000/auth/login         ← 直接訪問登入頁 (避開 / 的 @login_required redirect)
http://<SF-IP>:5000/api/health         ← API 健康檢查 (不需登入)
```

從 SF 主機本機測:
```powershell
Invoke-WebRequest http://localhost:5000/api/health -UseBasicParsing
```

## 故障排除

### Portal 顯示空白

跑診斷:
```powershell
.\diag.ps1
```

對應 Linux:
| 你看到 | Linux 等價 | 修法 |
|---|---|---|
| Service Stopped | `systemctl status portal → inactive` | 抓 wheels 重部署 |
| Port 5000 listen 127.0.0.1 | `ss -tlnp` 顯示 127.0.0.1 | 改 wsgi.py `host='0.0.0.0'` |
| HTTP 302 to /auth/login | `curl -v` 看 Location header | 直接訪問 `/auth/login` |
| HTTP 500 | Flask 內部錯 | 看 log |

### 改 wsgi.py listen 全介面

```powershell
$wsgi = 'C:\_portal\app\wsgi.py'
(Get-Content $wsgi) -replace "host='127.0.0.1'", "host='0.0.0.0'" | Set-Content $wsgi -Encoding UTF8
Restart-Service FileExchangePortal
```

對應 Linux:
```bash
sed -i "s/host='127.0.0.1'/host='0.0.0.0'/" /portal/app/wsgi.py
systemctl restart portal
```

### 看 Portal log

```powershell
# 設 NSSM 寫 log 到檔
& 'C:\Tools\nssm.exe' set FileExchangePortal AppStdout 'C:\_portal\logs\portal-stdout.log'
& 'C:\Tools\nssm.exe' set FileExchangePortal AppStderr 'C:\_portal\logs\portal-stderr.log'
Restart-Service FileExchangePortal

# Tail -f
Get-Content C:\_portal\logs\portal-stderr.log -Tail 50 -Wait
```

對應 Linux: `journalctl -u portal -f`

### sshd 啟動失敗 (套 v1.0.0.10 後仍 fail)

```powershell
# 驗 config 語法 (對應: sshd -t)
& 'C:\Program Files\OpenSSH\sshd.exe' -t -f 'C:\ProgramData\ssh\sshd_config'

# 看 Event Log (對應: journalctl -u sshd)
Get-WinEvent -LogName 'OpenSSH/Operational' -MaxEvents 20 |
    Format-Table TimeCreated, LevelDisplayName, Message -Wrap

# 還原備份 (對應: cp /etc/ssh/sshd_config.bak /etc/ssh/sshd_config)
ls C:\ProgramData\ssh\*.bak.*
Copy-Item C:\ProgramData\ssh\sshd_config.bak.<最新> C:\ProgramData\ssh\sshd_config -Force
Restart-Service sshd
```

## 附錄 A: 無外網替代方案

如果 SF 主機完全不能上 GitHub (corporate proxy 擋), 在外網 PC 抓 zip:

```powershell
# 外網 PC
mkdir C:\Temp\sf_install
cd C:\Temp\sf_install

# 抓 3 個檔
iwr https://github.com/alienid4/cl_ftp/raw/main/release-zip/sf-patch-v1.0.0.10.zip -OutFile p10.zip
iwr https://raw.githubusercontent.com/alienid4/cl_ftp/main/scripts/poc_setup_c_drive.ps1 -OutFile poc.ps1
iwr https://raw.githubusercontent.com/alienid4/cl_ftp/main/scripts/diagnose_portal.ps1 -OutFile diag.ps1

# 還要抓 wheels (在第 4 步用)
mkdir wheels; cd wheels
pip download flask waitress pyodbc python-ldap requests
cd ..

# 全部壓 zip
Compress-Archive -Path . -DestinationPath sf_install.zip
```

USB 拷到 SF 主機, 解壓後跑 Step 1 → Step 4 同樣指令。
