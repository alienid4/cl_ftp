| 項目 | 值 |
|---|---|
| **版本** | v2.1.0 |
| **日期** | 2026-05-20 |
| **對象** | RHEL 9 + 公司有 Red Hat Satellite |
| **耗時** | ~30 分鐘 (vs bundle 模式踩坑數天) |

---

## 前提條件

- [ ] SF 主機 RHEL 9 已建好 (IP 通內網)
- [ ] 公司 Red Hat Satellite 已部署且運作中
- [ ] **跟 IT / Satellite admin 拿 3 個值** (見下方)
- [ ] SF 主機能 https 訪問 Satellite

---

## Step 1: 跟 IT 拿 Satellite 資訊

寫信給 IT, 要這 3 個值:

```
主旨: 申請 SF 主機掛載 Satellite

說明:
  我們新部署一台 SF File Exchange Server (RHEL 9)
  需要從公司 Satellite 抓 RHEL 套件 (postgresql / nginx / samba / etc.)

請提供:
  1. Satellite URL (例: https://satellite.corp.local)
  2. Organization 名稱 (例: Default_Organization)
  3. Activation Key (請建一個 sf-server-key,
     訂閱 RHEL 9 BaseOS + AppStream)

SF 主機資訊:
  Hostname: <sf-host>
  IP: <sf-ip>
  OS: RHEL 9.6
  Role: File Exchange Server (內部用)
```

預期回覆:
```
SATELLITE_URL = https://satellite.corp.local
SATELLITE_ORG = Default_Organization
SATELLITE_KEY = sf-server-key
```

(Activation Key 設好後, SF 主機註冊就會自動拿到對應的 subscriptions + repos)

---

## Step 2: SF 主機一行部署

```bash
sudo SATELLITE_URL=https://satellite.corp.local \
     SATELLITE_ORG=Default_Organization \
     SATELLITE_KEY=sf-server-key \
     SF_ACCOUNTS=u01t \
     SF_PASSWORD='1qaz@WSX' \
     bash <(curl -fsSL https://github.com/alienid4/cl_ftp/raw/main/deploy-rhel/install_satellite.sh)
```

(把 URL / ORG / KEY 換成 IT 給的真實值)

或先抓再跑:

```bash
sudo curl -fsSL https://github.com/alienid4/cl_ftp/raw/main/deploy-rhel/install_satellite.sh \
    -o /tmp/install.sh
sudo SATELLITE_URL=... SATELLITE_ORG=... SATELLITE_KEY=... bash /tmp/install.sh
```

## 將來怎麼升級

### Code 升級 (SF 自己的程式)

```bash
cd /opt/sf
git pull
sudo ./deploy-rhel/install_all.sh
```

### 套件升級 (PostgreSQL / nginx 安全更新)

```bash
sudo dnf upgrade --security
# 或全升:
sudo dnf upgrade -y
```

→ 完全跟公司其他 RHEL 主機一樣的流程, IT 用 Satellite 集中管理。

## 故障排除

### `subscription-manager register` 失敗

```bash
# 看詳細錯誤
sudo subscription-manager register --org=X --activationkey=Y --force --verbose

# 常見:
# - SSL/TLS error → 確認 katello-ca-consumer rpm 裝對版本
# - Activation Key not found → 跟 admin 確認 key 名稱
# - Org name 大小寫敏感
```

### `dnf install` 找不到套件

```bash
# 看 enabled repo
sudo dnf repolist

# 應該看到:
# rhel-9-for-x86_64-baseos-rpms
# rhel-9-for-x86_64-appstream-rpms

# 沒看到 → Activation Key 沒掛這些 subscription, 跟 admin 確認 Content View 設定
```

### 主機已用 bundle 模式裝過, 要切回 Satellite?

```bash
# 1. 清理舊 bundle 安裝痕跡 (可選)
sudo dnf history list | head -20
sudo dnf history undo <id>  # 撤銷某次裝

# 2. 跑 Satellite install (idempotent)
sudo SATELLITE_URL=... bash install_satellite.sh
# 已裝的套件 skip, 沒裝的補裝
```
