# Runbook: 離線打包 + 內網 SF 主機部署

| 項目 | 值 |
|---|---|
| **版本** | v2.0.0.2 |
| **日期** | 2026-05-20 13:30 |
| **適用對象** | SF 主機內網隔離 (無外網), 需要先打包再帶進去 |
| **預期結果** | 不需要 SF 主機連 dnf / pypi, 純離線安裝 |
| **耗時** | 外網 PC 打包 10 分鐘 + SF 主機安裝 5 分鐘 |

---

## 前提

- [ ] **外網 RHEL PC** 一台 (公司給的 dev 機 / VM / 你自己的 Linux)
  - 要跟 SF 主機 OS 版本一致 (RHEL 8 vs 9 RPM 不通用)
- [ ] **USB / 內網 share** 把 tarball 拷進 SF 主機
- [ ] **SF 主機** (內網, 無外網) 已配好 IP + 防火牆基本能讓 SSH/RDP 進

---

## 流程圖

```
[外網 RHEL PC]                                  [SF 主機 (內網)]
                                                
1. clone repo                                   
2. ./build_offline_bundle.sh                    
   ↓ 抓 RPM + wheels + repo                    
   ↓ 打包 ~225 MB tarball              ──USB──► 3. 解壓 tarball
                                                4. ./install_offline.sh
                                                   ↓ 用本地 RPM 裝
                                                   ↓ 用本地 wheels 裝 Portal
                                                   ↓ 跑 install_all.sh
                                                5. 完成, 顯示訪問網址
```

## 段 1: 外網 RHEL PC 打包 (10 分鐘)

### 1.1 找一台外網 RHEL PC

要求:
- RHEL 8 或 9 (跟 SF 主機**同主版本**)
- 有 dnf 連得到公司 / 上游 mirror
- 至少 1 GB 可用空間 (下載暫存)

如果沒有 RHEL 外網 PC, 替代方案:
- Rocky Linux 9 / AlmaLinux 9 (RHEL clone, RPM 通用)
- 自己起個 RHEL 9 VM 在筆電上 (VirtualBox / VMware)
- 公司 IT 給個 dev VM

### 1.2 Clone repo + 打包

```bash
# 1. 裝 git (如果沒)
sudo dnf install -y git

# 2. clone
git clone https://github.com/alienid4/cl_ftp /tmp/sf
cd /tmp/sf

# 3. 確認最新 (含 v2.0.0.2 build script)
git pull
git describe --tags
# 應該看到 v2.0.0.2 或更新

# 4. 跑打包 (預設輸出 /tmp/sf-rhel-bundle)
sudo ./deploy-rhel/build_offline_bundle.sh

# 或指定輸出位置:
sudo ./deploy-rhel/build_offline_bundle.sh /home/me/sf-bundle
```

### 1.3 打包輸出

腳本跑完看到:
```
=== 完成 — Summary ===

輸出檔:
  /tmp/sf-rhel-bundle-20260520_1330.tar.gz (225M)
  /tmp/sf-rhel-bundle-20260520_1330.tar.gz.sha256
```

打包內容 (對齊 Linux 認知):

| 項目 | 對應 |
|---|---|
| `rpms/*.rpm` | `yum install` 用的套件 |
| `wheels/*.whl` | `pip install` 用的套件 |
| `repo/` | git clone 的 source |
| `install_offline.sh` | 給 SF 主機跑的一鍵腳本 |
| `README.md` | 給接手者看的說明 |

## 段 2: USB 拷貝到 SF 主機

### 2.1 拷到 USB

```bash
# 假設 USB mount 在 /mnt/usb
cp /tmp/sf-rhel-bundle-*.tar.gz /mnt/usb/
cp /tmp/sf-rhel-bundle-*.tar.gz.sha256 /mnt/usb/

# 卸載 USB
sync
umount /mnt/usb
```

或用 SCP 進中繼跳板:
```bash
scp /tmp/sf-rhel-bundle-*.tar.gz user@jumphost:/tmp/
```

### 2.2 在 SF 主機解壓

```bash
# 1. USB 插 SF 主機, mount
sudo mount /dev/sdb1 /mnt   # 視 USB 識別而定

# 2. 驗證 hash (對應 Linux sha256sum -c)
cd /mnt
sha256sum -c sf-rhel-bundle-20260520_1330.tar.gz.sha256
# 應該看到 "OK"

# 3. 拷到 /opt/install 解壓
sudo mkdir -p /opt/install
cd /opt/install
sudo tar xzf /mnt/sf-rhel-bundle-20260520_1330.tar.gz
cd sf-rhel-bundle/
ls
# README.md  install_offline.sh  repo/  rpms/  wheels/
```

## 段 3: SF 主機一鍵安裝 (5 分鐘)

```bash
# 在解壓目錄
sudo SF_ACCOUNTS=u01t SF_PASSWORD='1qaz@WSX' ./install_offline.sh
```

腳本內部會做:

| Step | 動作 | Linux 對照 |
|---|---|---|
| 1 | `dnf install --disablerepo='*' rpms/*.rpm` | 純離線裝 RPM |
| 2 | `cp -r repo/* /opt/sf/` | 拷 source code |
| 3 | `cp wheels/* /opt/sf/python_wheels/` | 拷 wheels (給 09 用) |
| 4 | `/opt/sf/deploy-rhel/install_all.sh` | 跑主部署流程 |

完成後看到:
```

                 http://10.92.198.16:5000/   ← 直連 Flask
```

## 驗證

### 1. 服務跑起來

```bash
systemctl is-active sshd nginx postgresql sf-portal
# 全部應該 active
```

### 2. 套件版本

```bash
# RPM
rpm -q postgresql-server nginx samba python3

# Python
sudo -u portal /opt/portal/venv/bin/pip list | grep -E 'flask|psycopg2'
```

### 3. SFTP 連線 (從別台主機測)

```bash
sftp u01t@<SF-IP>
# Password: 1qaz@WSX
sftp> ls
inbound  pending  outbound  archive  error
```

### 4. Portal HTTP (從別台 PC 開瀏覽器)

```
http://<SF-IP>/
```

### 5. 健康速查

```bash
sudo /opt/sf/deploy-rhel/health_check.sh
```

## 故障排除

### dnf install --disablerepo='*' 失敗 (dependency)

```bash
# 看缺什麼
sudo dnf install -y --disablerepo='*' rpms/*.rpm 2>&1 | grep 'No match'

# 如果缺 base RPM (例如 libxxx), 表示 SF 主機 OS 跟外網 PC 不同版本
# 解法: 用 RHEL 主機本身的 base repo
sudo dnf install -y rpms/*.rpm   # 不加 --disablerepo
# (需要 SF 主機能達 dnf base repo, 內網通常有 yum mirror)
```

### Python wheels 跟 system Python 版本不合

```bash
# 看 wheels 是哪個 Python 版本
ls /opt/sf/python_wheels/*.whl | head -3
# 例如 cp311 表示 Python 3.11

# 看 SF 主機 Python
python3 --version

# 不合的話, 在外網 PC 用對的 Python 重新下載:
python3.11 -m pip download -d wheels flask waitress ...
```

### Portal 跑不起來 (跟 v1.x 一樣)

跑 diagnose_portal.sh (還沒寫, 但我可以加):
```bash
sudo systemctl status sf-portal -l
sudo journalctl -u sf-portal -n 100
```

外網 PC:

```bash
cd /tmp/sf
git pull
sudo ./deploy-rhel/build_offline_bundle.sh
```

## 環境變數 (打包時可調)

build_offline_bundle.sh 目前用 hardcoded 套件清單。如果要客製:

```bash
# 編輯 build script, 修改 RPMS 陣列或 PIP_PACKAGES 陣列
vi /tmp/sf/deploy-rhel/build_offline_bundle.sh

# 例如想多加 fail2ban:
RPMS=(... fail2ban ...)
```

## 對應 Windows v1.x 工作流程

| 步驟 | Windows v1.x | RHEL v2.x |
|---|---|---|
| 外網 PC 抓套件 | `build_offline_bundle.ps1` 抓 .exe / .msi / .whl | `build_offline_bundle.sh` 抓 .rpm / .whl |
| 打包輸出 | `sf_offline_bundle_*.zip` (~350 MB) | `sf-rhel-bundle-*.tar.gz` (~225 MB) |
| SF 主機解壓 | `Expand-Archive sf_offline_bundle*.zip` | `tar xzf sf-rhel-bundle-*.tar.gz` |
| 一鍵安裝 | `.\install_offline.ps1` | `./install_offline.sh` |
| 一鍵部署 | (Step 9 跑 deploy/00-17) | (Step 4 跑 deploy-rhel/install_all.sh) |
| 驗證 | `health_check.ps1` | `health_check.sh` |

→ 概念完全一樣, 只是換語言。
