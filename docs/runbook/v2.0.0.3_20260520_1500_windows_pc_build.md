# Runbook: 從 Windows PC 打包 SF 離線安裝包 (內網無外網)

| 項目 | 值 |
|---|---|
| **版本** | v2.0.0.3 |
| **日期** | 2026-05-20 15:00 |
| **適用對象** | 你只有 Windows PC 能上網, SF 主機是 RHEL 內網 |
| **預期結果** | Windows PC 跑一個指令, 產出 ~225 MB tarball, 拷 USB 到 SF 主機 |
| **耗時** | 第一次 15 分鐘 (含 Docker 設定), 之後 5 分鐘 |

---

## 場景

```
[你的 Windows PC (有外網)]                  [SF 主機 (RHEL 內網, 無外網)]

  PowerShell 跑                                     
  build_bundle_windows.ps1                          
        ↓                                           
  Docker Desktop 起 Rocky Linux 9                   
  container, 在裡面跑 dnf download + git           
        ↓                                           
  輸出 sf-rhel-bundle-XXXXX.tar.gz                  
  (~225 MB)                            ──USB──►   解壓 + 跑 install_offline.sh
                                                        ↓
                                                  完成
```

## 一次性: Windows PC 裝 Docker Desktop

如果**已經裝過**, 跳過這段。

### 1. 下載

https://www.docker.com/products/docker-desktop/

(免費版即可, 個人用途無 license 問題)

### 2. 安裝

雙擊 .exe, 一路 Next。安裝完**重開機**。

### 3. 啟動 Docker Desktop

工作列右下角應該看到 Docker 圖示 (鯨魚)。等到不動才 ok。

### 4. 確認

打開 PowerShell, 試:
```powershell
docker version
docker pull hello-world
docker run --rm hello-world
```

看到 "Hello from Docker!" → ✅ 成功。

## 主要流程 (打包)

### 1. 開 PowerShell

不用系統管理員, 一般 PowerShell 即可。

### 2. 抓打包腳本

```powershell
# 切到任意目錄 (例如桌面)
cd $env:USERPROFILE\Desktop

# 抓 PS1 腳本
iwr https://raw.githubusercontent.com/alienid4/cl_ftp/main/scripts/build_bundle_windows.ps1 -OutFile build.ps1
```

### 3. 跑打包

```powershell
.\build.ps1
```

預期看到:
```
[1/4] 檢查 Docker...
[ok] Docker 已安裝且 daemon 跑著

[2/4] 準備輸出目錄...
[ok] 輸出目錄: C:\Users\xxx\Desktop

[3/4] Pull Rocky Linux 9 image (第一次 ~70 MB, 之後 cache)...
9.x.x: Pulling from library/rockylinux
Status: Downloaded newer image for rockylinux:9

[4/4] 在 Rocky 9 container 內打包...
=== Container 環境 ===
Rocky Linux release 9.x.x (Blue Onyx)

Cloning repo...

(dnf download 200+ MB...)
(pip download ~20 MB...)
(tar czf 打包...)

=== 完成 ===
sf-rhel-bundle-20260520_1500.tar.gz (225M)

下一步:
  1. 拷 .tar.gz 跟 .sha256 到 USB
  ...
```

整個過程約 5-10 分鐘 (取決於網速)。

### 4. 找輸出檔

```powershell
ls $env:USERPROFILE\Desktop\sf-rhel-bundle-*.tar.gz
```

看到一個 ~225 MB 的 .tar.gz, 跟一個 .sha256 校驗檔。

## USB 拷貝

### 拷到 USB

把 .tar.gz 跟 .sha256 拷到 USB 隨身碟。

### USB 接 SF 主機

```bash
# SF 主機 (RHEL)
# 1. 確認 USB device
lsblk
# 找到 USB (例如 /dev/sdb1)

# 2. mount
sudo mkdir -p /mnt/usb
sudo mount /dev/sdb1 /mnt/usb

# 3. 驗 hash (對應 Linux 你熟的)
cd /mnt/usb
sha256sum -c sf-rhel-bundle-*.sha256
# 看到 "OK"

# 4. 拷到 /opt/install
sudo mkdir -p /opt/install
cd /opt/install
sudo cp /mnt/usb/sf-rhel-bundle-*.tar.gz .

# 5. unmount USB
sudo umount /mnt/usb
```

---

## SF 主機解壓 + 一鍵安裝

```bash
cd /opt/install
sudo tar xzf sf-rhel-bundle-*.tar.gz
cd sf-rhel-bundle/

# 看內容
ls
# README.md  install_offline.sh  repo/  rpms/  wheels/

# 一鍵離線安裝
sudo SF_ACCOUNTS=u01t SF_PASSWORD='1qaz@WSX' ./install_offline.sh
```

跑完顯示訪問網址:
```
Portal HTTP : http://10.92.198.16/
SFTP        : sftp u01t@10.92.198.16   (密碼 1qaz@WSX)
```

```
                                                4. sudo ./install_offline.sh
                                                5. 訪問 http://<IP>/
```

## 故障排除

### Docker pull 卡住 / 失敗

公司可能擋 Docker Hub。試:

```powershell
# 用公司內 mirror
docker pull <corp-mirror>/rockylinux:9

# 或設代理
$env:HTTP_PROXY = "http://corp-proxy:8080"
$env:HTTPS_PROXY = "http://corp-proxy:8080"
docker pull rockylinux:9
```

### dnf download 在 container 內失敗

```powershell
# 進 container 互動 debug:
docker run --rm -it -v "${PWD}:/output" rockylinux:9 bash

# 在 container 內:
dnf install -y git
git clone https://github.com/alienid4/cl_ftp /tmp/sf
cd /tmp/sf
bash ./deploy-rhel/build_offline_bundle.sh /output/test
```

### 公司 PC 不能裝 Docker Desktop

**替代方案 A**: 用 WSL2 (Windows 內建, 不用裝 Docker)

```powershell
# 一次性裝 WSL + Ubuntu (有 dnf 可用)
wsl --install -d Ubuntu

# 重開後進 Ubuntu
wsl

# Ubuntu 內裝 dnf (Ubuntu 預設 apt, 要加 dnf)
sudo apt update
sudo apt install -y dnf git python3-pip

# 設 RHEL repo (Rocky Linux mirror)
sudo tee /etc/yum.repos.d/rocky.repo <<EOF
[rocky-baseos]
name=Rocky Linux 9 - BaseOS
baseurl=https://download.rockylinux.org/pub/rocky/9/BaseOS/x86_64/os/
enabled=1
gpgcheck=0

[rocky-appstream]
name=Rocky Linux 9 - AppStream
baseurl=https://download.rockylinux.org/pub/rocky/9/AppStream/x86_64/os/
enabled=1
gpgcheck=0
EOF

# 跑打包
git clone https://github.com/alienid4/cl_ftp /tmp/sf
cd /tmp/sf
sudo bash ./deploy-rhel/build_offline_bundle.sh /mnt/c/Users/<你>/Desktop/sf-bundle
```

**替代方案 B**: 用 VirtualBox + Rocky Linux ISO (純 GUI, 沒 Docker / WSL 也能跑)

```
1. 下載 VirtualBox (免費): https://www.virtualbox.org/
2. 下載 Rocky Linux 9 ISO: https://rockylinux.org/download
3. 起 VM, 裝 Rocky Linux (約 30 分鐘)
4. VM 內 clone repo + 跑 build_offline_bundle.sh
5. 共享資料夾把 tar.gz 帶出來
```

### 跑出來的 tar.gz 在 SF 主機解壓失敗

可能是 USB 傳輸壞掉, 比對 hash:

```bash
# SF 主機
sha256sum -c sf-rhel-bundle-*.sha256

# 不通過, 重拷
```

或 tar.gz 太大 USB 文件系統 (FAT32) 不支援 > 4 GB 單檔。
我們 bundle 才 ~225 MB, 不該遇到。除非 USB 異常。

## 為什麼用 Docker 而不是純 PowerShell

直接 PowerShell 抓 RPM:
- 每個 RPM 都要找完整 URL
- Dependency 樹要手動解 (例: postgresql 依賴 libpq, libicu, ...)
- 200+ 個檔, 手動列不現實

Docker:
- 一個 `dnf download --resolve --alldeps` 解所有 dependency
- 跟 RHEL 環境 100% 一致 (Rocky = RHEL clone, RPM 通用)
- 一次性投資 (裝 Docker), 之後重打包都快

---

## 為什麼用 Rocky Linux 不直接用 RHEL

- RHEL container image 要 Red Hat subscription
- Rocky Linux 9 是 RHEL 9 的免費 clone
- 所有 RPM 100% 通用 (相同 binary)
- Docker Hub 直接可拉, 不用設 subscription

---

## 將來重打包 (新 patch 出來時)

```powershell
# Windows PC
.\build.ps1   # 自動拉最新 git
```

每次跑都會 pull 最新 repo, 含最新 patch。
