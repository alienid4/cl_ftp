## 一鍵部署 (Linux 對照: `./setup.sh`)

```bash

git describe --tags --abbrev=0
```

## 驗證 (Linux 對照: 健康檢查)

### 1. 看服務 (對應 `systemctl status`)

```bash
sudo systemctl status sshd nginx postgresql sf-portal --no-pager | grep Active
# 預期: 全部 active (running)
```

### 2. 看 port (對應 `ss -tlnp`)

```bash
ss -tlnp | grep -E ':(22|80|5000|5432)\b'
# 預期看到:
# 22 (sshd), 80 (nginx), 5000 (gunicorn), 5432 (postgres)
```

### 3. 看 u01t 帳號

```bash
id u01t
# uid=1xxx(u01t) gid=1xxx(sftp_users) groups=1xxx(sftp_users)

ls -la /data/exchange/u01t/
# drwxr-xr-x  root  root      .          ← chroot 點
# drwxrws---  u01t  sftp_users inbound   ← 可寫
# drwxrws---  u01t  sftp_users pending
# drwxrws---  u01t  sftp_users outbound
# drwxrws---  u01t  sftp_users archive
# drwxrws---  u01t  sftp_users error
```

### 4. SFTP 連線測試 (從別台 PC 跑)

```bash

sftp> ls
# inbound  pending  outbound  archive  error
sftp> cd inbound
sftp> put /tmp/test.txt

# test.txt
sftp> quit
```

Windows PC 用 WinSCP / FileZilla:
- Host: <SF-IP>
- Port: 22
- User: u01t
- Password: 1qaz@WSX
- Protocol: SFTP

## 故障排除

### Portal 空白

```bash
# 看 log (對應 journalctl -u portal)
sudo journalctl -u sf-portal -n 50

# 看 nginx error
sudo tail -50 /var/log/nginx/sf-portal-error.log

# 重啟
sudo systemctl restart sf-portal nginx
```

### SFTP 連不上 / 認證失敗

```bash
# 看 sshd log
sudo journalctl -u sshd -n 50 | grep u01t

# 驗 sshd config 語法
sudo sshd -t

# 重設密碼
echo 'u01t:1qaz@WSX' | sudo chpasswd

# 重啟 sshd
sudo systemctl restart sshd
```

### SFTP 連上但卡 chroot 錯誤

```
"fatal: bad ownership or modes for chroot directory"
```

→ chroot 點權限不對, 應該 `root:root 0755`:

```bash
sudo chown root:root /data/exchange/u01t
sudo chmod 0755 /data/exchange/u01t
sudo systemctl restart sshd
```

### DB 連不上

```bash
# 看 postgres 狀態
sudo systemctl status postgresql

# 測連線
sudo -u postgres psql -c "SELECT 1;"
sudo -u portal psql -h localhost -d file_exchange_audit -c "\dt"
```

## 之後加更多帳號

PoC 通過, 正式要建 u01/u02/u03 時:

```bash
# 一次建 3 個帳號 (密碼隨機產生)
for acct in u01 u02 u03; do
    pw=$(openssl rand -base64 24)
    sudo SF_ACCOUNTS="$acct" SF_PASSWORD="$pw" \
        ./deploy-rhel/01_setup_directories.sh
    sudo SF_ACCOUNTS="$acct" SF_PASSWORD="$pw" \
        ./deploy-rhel/02_setup_ownership.sh
    sudo SF_ACCOUNTS="$acct" SF_PASSWORD="$pw" \
        ./deploy-rhel/04_create_sftp_accounts.sh
    echo "建立 $acct, 密碼: $pw"   # 記下來給 PAM
done

# 對應 DB seed (Portal 後台會用)
sudo -u portal psql -d file_exchange_audit <<EOF
INSERT INTO business_code (code, dept, description, samba_target, allowed_proto)
VALUES
    ('u01', 'HR',  'HR 部門檔案交換', '/data/exchange/samba/hr', 'SFTP'),
    ('u02', 'FIN', 'Finance 部門檔案交換', '/data/exchange/samba/finance', 'SFTP'),
    ('u03', 'OPS', 'Operations 部門檔案交換', '/data/exchange/samba/security', 'SFTP');
EOF
```

## 之後拿到憑證 (補 HTTPS)

```bash
# 1. 申請憑證, 拿到 .crt + .key
sudo cp sf.crt /etc/pki/tls/certs/
sudo cp sf.key /etc/pki/tls/private/
sudo chmod 600 /etc/pki/tls/private/sf.key

# 2. 取消 nginx HTTPS 區塊的註解
sudo vi /etc/nginx/conf.d/sf-portal.conf
# 找到 "HTTPS (拿到憑證後啟用)" 段, 移除 # 開頭

# 3. 重載 nginx
sudo nginx -t && sudo systemctl reload nginx

# 4. 防火牆放行 443
sudo firewall-cmd --permanent --zone=sf --add-service=https
sudo firewall-cmd --reload
```

之後訪問 https://<SF-IP>/ 走 TLS。

## 重要環境變數參考

跑 install_all.sh 前可帶:

```bash
sudo \
  SF_ACCOUNTS="u01t" \                # 帳號 (空格分隔多個)
  SF_PASSWORD='1qaz@WSX' \            # 統一密碼 (建議只在 PoC 用)
  SF_DATA_ROOT=/data/exchange \       # 業務檔位置
  SF_PORTAL_ROOT=/opt/portal \        # Portal 系統檔
  SF_PORTAL_PORT=5000 \               # Flask 內部 port
  SF_DB_NAME=file_exchange_audit \
  SF_DB_USER=portal \
  SF_SKIP_AD=1 \                      # 1=跳過 AD, 0=接 AD
  SF_SKIP_SAMBA=0 \                   # 1=跳過 Samba
  SF_CORP_NET=10.0.0.0/8 \           # 公司內網段 (防火牆白名單)
  SF_NTP_SERVERS="ntp1.corp.local" \
  SF_ALERT_MAIL=it-admin@corp.local \
  ./deploy-rhel/install_all.sh
```

## 對應 Linux 思維

| 你會的 Linux 做法 | 在這個專案的對應 |
|---|---|
| `useradd -s /sbin/nologin u01t` | `04_create_sftp_accounts.sh` 已做 |
