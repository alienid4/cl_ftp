# Runbook: 手動下載 EPEL Python RPM (v2.2.0)

| 項目 | 值 |
|---|---|
| **版本** | v2.2.0 |
| **日期** | 2026-05-21 |
| **對象** | SF 主機禁 EPEL + SF 完全離線 + PC 網路受限 |
| **目的** | 手動下載 Portal 需要的 EPEL Python RPM, 不靠任何自動化 |

---

## 為什麼要這條路 (對齊鐵律 11)

USER 環境限制:
1. **SF 主機完全不能上網** — 連 github / EPEL mirror 都不通
2. **PC 網路也很多限制** — 不能跑 docker, GitHub Actions debug 卡住
3. **公司禁 EPEL repo** — 但允許「外部下載 RPM → 拷進來」這條路

唯一能走: PC 用瀏覽器點連結下載 RPM → 打成 tar → scp 到 SF → SF dnf install local.

## 完整步驟

### Step 1: PC 用瀏覽器點 7 個連結下載

把全部 7 個 .rpm 下載到同一資料夾 (建議 `C:\Temp\epel-rpms\`):

```
C:\Temp\epel-rpms\
├── python3-flask-2.0.3-1.el9.1.noarch.rpm
├── python3-werkzeug-2.0.3-3.el9.1.noarch.rpm
├── python3-gunicorn-21.2.0-2.el9.noarch.rpm
├── python3-itsdangerous-2.0.1-4.el9.noarch.rpm
├── python3-click-8.0.3-2.el9.noarch.rpm
├── python3-blinker-1.5-1.el9.noarch.rpm
└── python3-ldap3-2.8.1-4.el9.noarch.rpm
```

> 如果 `dl.fedoraproject.org` 被擋, 用 `curl.exe -sI <URL>` 確認回 200 OK; 不通的話跟 IT 申請 whitelist 或改用 mirror.

### Step 2: PC 打成 tar

```powershell
cd C:\ClaudeHome\SFTP
.\pack_local_rpms.ps1 -SrcDir C:\Temp\epel-rpms
```

輸出: `release-zip\sf-epel-pyrpms.tar.gz` (~1.7 MB)

腳本會做:
- 驗 7 個必要套件都齊
- 寫 README.txt + MANIFEST.txt
- 打 tar.gz + sha256

### Step 3: PC scp 推到 SF

```powershell
.\xfer_to_sf.ps1 -SfHost <SF-IP>
```

腳本會做:
- `ssh mkdir -p /opt/sf/release-zip`
- `scp release-zip/* → SF:/opt/sf/release-zip/`
- SF 端 `sha256sum -c` 驗檔完整
- 印下一步指令

### Step 4: SF 跑 fix_portal.sh (完全離線)

```bash
ssh root@<SF-IP> 'sudo bash /opt/sf/release-zip/latest-fix-portal.sh'
```

fix_portal.sh 自動:
- Step 1a: `dnf install` RHEL AppStream Python (走 Satellite)
- Step 1b: 讀 `/opt/sf/release-zip/sf-epel-pyrpms.tar.gz` → 解壓 → `dnf install --disablerepo='*' --nogpgcheck rpms/*.rpm`
- Step 2: cp `/opt/sf/portal/*` → `/opt/portal/app/`
- Step 3: 建 portal DB user + appsettings.json
- Step 4: 寫 `/etc/systemd/system/sf-portal.service` (用 gunicorn)
- Step 5: `firewall-cmd --add-service=http`
- Step 6: `systemctl enable --now sf-portal` + 30s retry 驗 HTTP

跑完成功印:
```
✅ Portal 已修好, 可給 USER 測試
Portal: http://<IP>/        (nginx 反代 80 → 5000)
Direct: http://<IP>:5000/   (本機 debug)
```

## 將來怎麼升級

### EPEL 套件升級

EPEL 出新版時 (例如 flask 2.0.3 → 2.3.0):

1. 看 dl.fedoraproject.org 找新版本 RPM URL
2. 更新本文件的 7 個連結
3. 重跑 Step 1-4

(或 GitHub Actions workflow 修好之後自動產, 詳見 `docs/dev-log/skill_sf_workflow.md` 鐵律 10)

## 故障排除

### 點連結 404

- EPEL 出新版舊版本可能被砍, 用 https://archives.fedoraproject.org/pub/archive/epel/9/Everything/x86_64/Packages/ 找
- 或本文件版本太舊, 跟 Claude 講重新查

### pack_local_rpms.ps1 抱怨「缺套件」

- 7 個都下載齊了嗎? 檢查資料夾 .rpm 數量
- 命名有沒有改 (例如 `(1).rpm` suffix)? 移掉

### xfer_to_sf.ps1 ssh 失敗

- `ssh-keygen` 產 key 後 `ssh-copy-id root@<SF-IP>` (要 SF 允許 password auth 一次)
- 或加 `-i <key>` 參數

### SF dnf install --disablerepo='*' --nogpgcheck *.rpm 失敗

最可能: GPG signature 不過 (用 `--nogpgcheck` 已經跳過)
其次: 依賴沒齊, 看錯誤訊息哪個 lib 缺, 從 RHEL AppStream 補裝

```bash
# 看缺啥
dnf install -y --disablerepo='*' --nogpgcheck /opt/sf/release-zip/sf-epel-pyrpms-extracted/rpms/*.rpm 2>&1 | grep -i "needed\|require\|missing"
```
