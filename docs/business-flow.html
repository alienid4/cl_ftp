Phase 1 — 上傳階段

耗時: 數秒 ~ 30 秒
1 PRD AP 主機申請 PAM 取得 u01 密碼 — 操作: AP 系統 / 上傳人 (張小明)
上傳前先到 PAM 系統申請 u01 帳號的密碼(限時取用、留稽核軌跡)。
系統PAM 輸出u01 臨時密碼 + PAM request ID (供 chain_id 串接) 階段⏳ 階段二接 PAM
2 SFTP 上傳到 SF — AP 系統 → SF:22
AP 用 u01 + 密碼,sftp 上傳 N 個檔到 /data/exchange/u01/inbound/。
指令sftp u01@SF 然後 put 月結_*.csv chrootu01 看不到 u02/u03/系統檔 事件每個 put OpenSSH log 寫一筆 階段⏳ v2.5.1 SFTP 設定待跑
[OpenSSH log] user=u01 src=10.20.5.31 file=月結_客戶清單.csv size=2.1MB action=upload
3 排程 batch_aggregator 偵測新檔 — 背景 systemd timer, 每 5 秒掃一次
掃 /data/exchange/u0X/inbound/ 新檔,套滑動 30 秒邏輯聚合成批次。
規則同 u0X + 同來源 IP + 30s 內 → 同一批 安全閥超過 5 分鐘強制 close (新檔開新批) DB 變化INSERT INTO batch + batchfile 階段⏳ v2.5.2 待寫排程
[batch] batch_id=u01-20260520-1432 file_count=5 total_size=8.4MB status=PENDING_APPROVAL [batchfile x5] 每檔一筆, decision=NULL (待簽)
4 Portal 寫 AuditLog + 寄通知信 — Flask 程式 + SMTP
Portal 偵測新 batch (排程觸發 callback),寫 AuditLog 並寄 mail 給該業務的 5 位簽核人。
事件BATCH_PENDING + 5 封 mail 收件人AD 群組 g_u01_approvers 所有成員 階段⏳ SMTP 待設
[auditlog] event_type=BATCH_PENDING batch_id=u01-20260520-1432 cef_signature=SF-210 [mail x5] 主旨: "[SF] u01 待您簽核 (批次 1432, 5 檔)"

Phase 2 — 簽核階段 (ANY 制)

耗時: 數分鐘 ~ 7 天 (超時自動駁回)
5 簽核人開瀏覽器 → AD 登入 Portal — 王主管 / 林副理 / 黃組長 / 周稽核 / 吳資安 (任一人)
收到通知信,點連結進 Portal,用個人 AD 帳號登入。
URLhttps://sf.corp/auth/login 認證LDAP bind 到公司 AD (or 階段一 DEV_MODE 任意) 看到首頁 KPI 有「待我簽核 1 件」 階段⏳ AD 待接
[auditlog] event_type=LOGIN_OK actor=CORP\wang.manager source_ip=10.30.1.88 [portaluser] UPSERT (cache 顯示名 / 部門 / 群組)
6 點「我的待簽」→ 看到批次 + 展開檔案 — Portal 一行 1 批 list
Portal 列我有權簽的批次 (依 AD 群組過濾)。點批次「展開」看內含 5 個檔 + SHA-256 + 大小 + 用途說明。
過濾WHERE status='PENDING_APPROVAL' AND g_u01_approvers in user.groups UI每批顯 [全收] [全退] [細選] 三按鈕
7 決策: ANY 制 (任 1 同意即放行, 任 1 駁回即駁回) — 行 row-level lock 競態處理

✅ 7a 全收 (任 1 同意)

UPDATE batch SET status='APPROVED', approved_by=自己
所有檔自動搬到 samba
AuditLog: APPROVE_OK + MOVE_OK x5

⚠️ 7b 細選 (3 收 2 退)

UPDATE batchfile SET decision='APPROVED'/'REJECTED' per file
收的搬 samba, 退的進 error/
batch.status='APPROVED_PARTIAL'

❌ 7c 全退 (任 1 駁回 + 寫原因)

UPDATE batch SET status='REJECTED', reject_reason=...
全部檔進 /error/, 上傳人收 mail
AuditLog: APPROVE_REJECT
競態: 二人同時點同意 → DB row-level lock 先到先處理, 後到者 UI 顯「已由 XXX 處理」。一同意 + 一駁回 → 同上,先到先處理。
8 Portal 搬檔到 OA samba 目錄 — Flask 用 svc_portal 服務帳號
shutil.move(/data/exchange/u01/inbound/月結_客戶清單.csv → /data/exchange/samba/architecture/)
NTFS ACLsamba 目錄給 dept_archi_view 群組讀取權 SACL啟用檔案存取稽核 → SMB 下載事件 DB 變化UPDATE batchfile SET final_path=..., moved_at=...
[auditlog x5] event_type=MOVE_OK target_path=/data/exchange/samba/architecture/月結_客戶清單.csv
9 寄通知信給 OA 端可下載群組 — SMTP
通知 dept_archi_view AD 群組所有成員 "u01 新檔可下載"。
訊息含: 檔名清單 / 來源業務代號 / 放行人 / samba 路徑 / 7 天到期日。

⚠️ 異常處理路徑

場景 A — SFTP 上傳被擋: 來源 IP 不在白名單 → 22 port 拒絕 → AP 端收 connection refused.
→ 解: IT 加 firewall 規則 firewall-cmd --add-rich-rule.
場景 B — 簽核人都不在 (連假): 7 天超時 → 自動駁回 + 通知上傳人重送.
→ 解: Portal 可設代簽人 (未實作), 或業務負責人加新成員到 g_u01_approvers.
場景 C — 大檔上傳超過 5 分鐘安全閥: 後續新檔開新批 → 同個檔可能分散到 2 批簽.
→ 解: 安全閥可調 (5/10/15 分鐘), 或約定 AP 端分小段傳.
場景 D — 簽核通過但搬檔 fail (samba 滿 / NTFS 權限錯): MOVE_FAIL 寫 AuditLog + 告警 IT.
→ 解: 檔留 inbound/, IT 處理後手動觸發重搬.
場景 E — OA USER 收信但 samba 掛不上: AD 群組未生效 / 跨網段防火牆問題.
→ 解: Portal「我的可下載」改走 HTTPS ZIP, 100% 通.
