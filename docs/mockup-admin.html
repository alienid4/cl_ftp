⚠️ 確認執行: 此操作會自動執行下列步驟

  1. 在 SF 上建立新實體目錄 D:\DataExchange\samba\arch_new\
  2. 套用 NTFS ACL: 給 CORP\dept_archi_view 讀取權, 給 svc_portal 寫入權
  3. 建立 SMB Share: New-SmbShare -Name "arch_new"
  4. 啟用檔案存取稽核 (SACL) → Windows Security Event Log
  5. 更新 Portal DB: u01.target_samba = \\SF\arch_new\
  6. 寫入 AuditLog: SAMBA_PATH_CHANGED
⚠ 副作用:
  • 舊路徑 \\SF\architecture\ 不會立即刪除 — 已放在裡面的檔案還在 7 天倒數
  • 新上傳的 u01 檔案會自動進入新路徑
  • OA 端 USER 必須重新掛載 SMB 才能看到新路徑
  • 舊路徑檔案 7 天到期後自動清空, 空殼系統再自動刪
  • 歷史 AuditLog 不會被改寫 (稽核要求)
