典型流程: SFTP 上傳 → 簽核 → SMB 下載

[PRD AP 系統] ─① sftp u01@SF / 上傳檔 ─► [SF :22 OpenSSH] │ ▼ /data/exchange/u01/inbound/ │ ② 排程偵測新檔 (滑動 30s) ▼ INSERT INTO batch + batchfile │ ③ mail 通知 g_u01_approvers ▼ [簽核人個人 PC] ◄─ HTTP :80 ─► [SF nginx → gunicorn :5000 → PostgreSQL :5432] │ ④ 點「全收」 (Portal) ▼ mv /data/exchange/u01/inbound/xxx → /data/exchange/samba/architecture/ │ ⑤ AuditLog 寫入 ▼ [OA 端 USER] ─⑥ \\SF\architecture\─► [SF :445 Samba] (還沒裝) 或 Portal 批次 ZIP (Portal 端代下)
目前進度: 第 ③ ④ ⑤ 步 (Portal 簽核 + DB 寫入) 已可走 dev 流程, 但第 ① ② ⑥ 步 (SFTP 進來 / 排程 / SMB 出去) 還沒接, 所以「上下游」沒進來,功能跑空轉。

下一步 (依優先序)

🚦 接資料源優先序

1️⃣
SFTP 上傳 (最優先) — 沒這個簽核流程跑不起來。
動作: 建 u01~u04 本機帳號 + chroot 設定 + sshd_config + firewall 放 22。
PAM 後接 (先用固定密碼測流程)。
2️⃣
排程 batch_aggregator — 沒這個, 上傳了 Portal 也看不到。
動作: 寫 systemd timer 每 5 秒掃 /data/exchange/u0X/inbound/, 滑動 30s 聚合進 batch 表。
3️⃣
AD 整合 (DEV_MODE 關掉) — 沒這個只能用 dev 帳密。
動作: 開單給 AD 管理員 (見 v2.4.4_ad-integration-sop.md), 拿到 AD_SERVER + 服務帳號後改 .env, restart sf-portal。
4️⃣
SMB 下載端 — 沒這個 OA USER 拿不到檔。
動作: dnf install samba, 設 share /data/exchange/samba/, 綁 dept_xxx_view AD 群組。
替代: Portal 內建「批次 ZIP 下載」(USER 走 HTTP 拿, 不用 SMB).
5️⃣
HTTPS + 公司憑證 — 改 http://10.92.198.14/ 為 https://sf.corp/
動作: 申請 cert, nginx config 加 listen 443 ssl, 防火牆放 443。
6️⃣
SMTP 通知信 — 沒這個簽核人不知道有檔要簽。
動作: .env 內 SMTP_SERVER 換公司 mail relay IP, 寫測試信驗通。
7️⃣
NTP 同步 — AuditLog 時間要跟外部對得起來。
動作: chronyc add server <公司NTP> iburst。
8️⃣
PAM 整合 (階段二) — u01~u04 密碼 PAM 自動 rotate。
階段一可以先固定密碼 (PAM 還沒接得上 SF 部分)。
9️⃣
SIEM forwarder (階段二) — AuditLog 推到公司 SIEM。
階段一 AuditLog 留在 PG 自查就好。
