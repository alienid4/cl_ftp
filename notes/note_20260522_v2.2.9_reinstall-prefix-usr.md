# Note 2026-05-22 v2.2.9 — 重裝 wheels 到 /usr (RHEL 標準位置)

## 確認

USER chmod 後仍失敗:
```
sudo -u nginx python3 -c "import flask_login"
→ ModuleNotFoundError
```

原因: RHEL 9 系統 python3 預設 `sys.path` 不包含 `/usr/local/lib/python3.9/site-packages/`
(只有 root user 因為某些原因看得到, nginx user 走標準 sys.path)

解法: 重裝到 `/usr/lib/python3.9/site-packages/` (在預設 sys.path 內)

## 修法 (3 步, 在 SF 上跑)

### 1. 先驗證 sys.path 差異 (確認假設)

```
echo "=== root sys.path ==="
python3 -c "import sys; print('\n'.join(sys.path))"

echo "=== nginx sys.path ==="
sudo -u nginx python3 -c "import sys; print('\n'.join(sys.path))"
```

預期: 兩者其實一樣, 但 `/usr/local/lib/python3.9/site-packages/` **不在** 任一個的 sys.path。

(root 之前能 import 是個謎, 可能是 root home 有 .pth 檔或 PYTHONPATH 環境變數)

### 2. 解除舊安裝 + 重裝到 /usr

```
pip3 uninstall -y Flask-Login flask-session cachelib

cd /tmp/ftp-lab

pip3 install --no-index --no-deps --prefix=/usr Flask_Login-0.6.3-py3-none-any.whl
pip3 install --no-index --no-deps --prefix=/usr cachelib-0.10.2-py3-none-any.whl
pip3 install --no-index --no-deps --prefix=/usr flask_session-0.5.0-py3-none-any.whl

# 驗
ls -la /usr/lib/python3.9/site-packages/flask_login/ 2>&1 | head -3
ls -la /usr/lib/python3.9/site-packages/flask_session/ 2>&1 | head -3
ls -la /usr/lib/python3.9/site-packages/cachelib/ 2>&1 | head -3
```

(`--prefix=/usr` → 套件落到 `/usr/lib/python3.9/site-packages/`)

### 3. nginx user 驗 + 重啟 sf-portal

```
sudo -u nginx python3 -c "from flask_login import LoginManager; from flask_session import Session; import cachelib; print('OK')"

systemctl restart sf-portal
sleep 3
systemctl status sf-portal --no-pager | head -10
ss -tlnp | grep 5000
curl -s -o /dev/null -w "HTTP: %{http_code}\n" http://localhost:5000/
```

預期:
- `OK`
- `Active: active (running)`
- Port 5000 LISTEN
- HTTP 200 / 302 / 401 / 403

## 如果 Step 2 的 pip install --prefix=/usr 失敗

可能 pip3 不認 --prefix, 改用 --target:

```
pip3 install --no-index --no-deps --target=/usr/lib/python3.9/site-packages Flask_Login-0.6.3-py3-none-any.whl
pip3 install --no-index --no-deps --target=/usr/lib/python3.9/site-packages cachelib-0.10.2-py3-none-any.whl
pip3 install --no-index --no-deps --target=/usr/lib/python3.9/site-packages flask_session-0.5.0-py3-none-any.whl
```

(`--target` 比 `--prefix` 更直接, 直接指定目錄)

## 如果還是不行

再回報, 看 sudo -u nginx 的 sys.path 完整輸出 + 重裝後的目錄 ls -la。可能還有 SELinux 影響, 那就要看 `ausearch -m AVC --start recent` 抓 SELinux denial。
