# Note 2026-05-22 v2.3.4 — Snapshot 回去後一次裝完

## 適用情境

USER snapshot 回到 **RHEL 9 + Satellite 註冊** 的乾淨狀態, 要重新裝 Portal。

## 前提條件 (snapshot 必須具備)

- [x] RHEL 9.x
- [x] Satellite 註冊好 (能 `dnf install` 從公司 mirror)
- [x] 網路通公司內網 (能 ssh / scp 從 PC)
- [x] **不需要外網 (本腳本不 curl github / pypi)**

如果 snapshot 沒符合上述, 先跑 `install_satellite.sh` 註冊 Satellite。

## 你做的 3 件事

### 1. PC 準備好以下檔案 (任一目錄都行, 例如 D:\sf-deploy\)

| 類型 | 數量 | 來源 / 連結 |
|---|---|---|
| install-portal-all-in-one-v2.3.4.sh | 1 | https://github.com/alienid4/cl_ftp/raw/main/release-zip/install-portal-all-in-one-v2.3.4.sh |
| EPEL RPMs (python3-flask 等) | 7 | https://github.com/alienid4/cl_ftp/blob/main/docs/runbook/v2.2.0_20260521_epel_pyrpms_manual.md |
| PyPI wheels (Flask-Login 等) | 3 | https://github.com/alienid4/cl_ftp/blob/main/notes/note_20260522_v2.2.7_pip-wheels.md |
| portal/ source 整個目錄 | 1 | 你 PC 的 `C:\ClaudeHome\SFTP\portal\` (git pull 拿到的) |

### 2. WinSCP 全部丟到 SF /tmp/ftp-lab/

## v2.3.4 流程 (9 步)

| 步驟 | 內容 | 跨 snapshot 狀態? |
|---|---|---|
| 0 | env check RHEL 9 | 通用 |
| 1 | 找 RPM/wheel/portal | 通用 |
| 2 | dnf install **nginx + postgresql + python deps + firewalld** | ✨ idempotent |
| 2a | postgresql-setup --initdb (if 沒) + enable/start | ✨ idempotent |
| 2b | enable/start nginx + firewalld | ✨ idempotent |
| 3 | rpm -Uvh --force --nodeps EPEL Python | idempotent |
| 4 | unzip wheel 到 /usr/lib/python3.9/site-packages + chmod | idempotent |
| 5 | rm -rf /opt/portal/app/ 再 cp portal/ | idempotent |
| 6 | DB + portal user + appsettings.json + pg_hba | idempotent |
| 7 | 寫 sf-portal.service (gunicorn) | idempotent |
| 8 | 寫 /etc/nginx/conf.d/sf-portal.conf 反代 80→5000 + SELinux setsebool | ✨ idempotent |
| 9 | systemctl start sf-portal + 30s retry 驗 HTTP + nginx 80 驗 | 通用 |

✨ = v2.3.4 新加的 (相比 v2.3.3)

## 失敗時看哪

腳本自動 dump:
- systemctl status sf-portal
- journalctl 最後 20 行
- /opt/portal/logs/portal-stderr.log 最後 30 行
- 手動 debug gunicorn 完整指令
