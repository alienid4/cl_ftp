# Note 2026-05-22 v2.2.8 — pip 裝在 root 看得到 / nginx 看不到的位置

## 情境

USER 跑 v2.2.7 pip install 完, 驗 import:
- `python3 -c "from flask_login import LoginManager"` → OK (root user)
- gunicorn 跑 (走 nginx user 643) → `ModuleNotFoundError: No module named 'flask_login'`

代表 pip3 把套件裝到 **root 看得到、nginx 看不到** 的路徑。

最可能: 裝到 `/root/.local/lib/python3.9/site-packages/` (pip 預設 user install fallback)

## 4 步診斷 + 修正

### 1. 找 flask_login 真實安裝位置

```
python3 -c "import flask_login; print(flask_login.__file__)"
find / -name 'flask_login' -type d 2>/dev/null | head -5
```

可能輸出 (對應問題):
| 輸出位置 | 問題 |
|---|---|
| `/root/.local/lib/python3.9/site-packages/flask_login/` | ✗ 只有 root 看得到 |
| `/usr/local/lib/python3.9/site-packages/flask_login/` | ✓ 系統全域 (但可能權限不對) |
| `/usr/lib/python3.9/site-packages/flask_login/` | ✓ RHEL 套件位置 |

### 2. 確認 nginx user 能否 import

```
sudo -u nginx python3 -c "import flask_login; print('nginx OK:', flask_login.__file__)"
```

| 輸出 | 意義 |
|---|---|
| `nginx OK: ...` | ?? 不該失敗才對, 看 gunicorn 是不是用別的 python |
| `ModuleNotFoundError` | 確定 nginx 看不到 |

### 3. (確定問題後) 重裝到全域位置

如果 Step 1 顯示在 `/root/.local/...`, 移到 `/usr/local/`:

```
pip3 uninstall -y Flask-Login flask-session cachelib

# 重新安裝指定全域位置
cd /tmp/ftp-lab
pip3 install --no-index --no-deps --target=/usr/local/lib/python3.9/site-packages Flask_Login-0.6.3-py3-none-any.whl
pip3 install --no-index --no-deps --target=/usr/local/lib/python3.9/site-packages cachelib-0.10.2-py3-none-any.whl
pip3 install --no-index --no-deps --target=/usr/local/lib/python3.9/site-packages flask_session-0.5.0-py3-none-any.whl
```

或更標準的 `--prefix`:
```
pip3 install --no-index --no-deps --prefix=/usr Flask_Login-0.6.3-py3-none-any.whl
pip3 install --no-index --no-deps --prefix=/usr cachelib-0.10.2-py3-none-any.whl
pip3 install --no-index --no-deps --prefix=/usr flask_session-0.5.0-py3-none-any.whl
```

(`--prefix=/usr` 裝到 `/usr/lib/python3.9/site-packages/`, 跟 dnf 裝的位置一致)

### 4. 驗證 + 重啟

```
sudo -u nginx python3 -c "from flask_login import LoginManager; print('OK')"
systemctl restart sf-portal
sleep 3
systemctl status sf-portal --no-pager | head -8
ss -tlnp | grep 5000
```

預期:
- `Active: active (running)`
- `LISTEN 0 ... 127.0.0.1:5000 ...`

---

## Step 1 + Step 2 輸出貼回 Claude 確認方向, 再跑 Step 3

如果 Step 1 顯示在 /root/.local/, 確定走 `--prefix=/usr` 重裝。
如果在別處, Claude 給對應修法。
