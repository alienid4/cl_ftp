# Note 2026-05-22 v2.3.2 — 重來: all-in-one 一次裝完

## 為什麼重寫

之前 8 次 iteration (v2.2.0 → v2.3.1) 每次只修一個錯誤. 現在把所有 fix 整合一支 `install_portal_all_in_one.sh`, USER 一次就裝完.

## 對齊規範 (從前面學到的)

- ✅ 公司禁 pip → 用 `unzip` 解 wheel
- ✅ 公司禁 EPEL → RPM 手動下載 scp 來
- ✅ SF 不能上網 → 全本地檔, 不 curl github
- ✅ 散檔 *.rpm 過濾重複 "(1).rpm"
- ✅ 補 RHEL AppStream 依賴 (jinja2 / packaging / pyasn1)
- ✅ gunicorn binary 動態抓 (/usr/bin/gunicorn 不是 gunicorn-3)
- ✅ DB CREATE 拆出 DO block (postgres 限制)
- ✅ unzip 到 /usr/lib/python3.9/site-packages (在 nginx sys.path)
- ✅ wsgi.py 自動 patch module-level app

## 你只要做 3 件事

### 1. PC 準備好以下檔案 (應該之前都下載了)

| 類型 | 數量 | 來源 |
|---|---|---|
| EPEL RPM | 7 個 | dl.fedoraproject.org (note v2.2.0) |
| PyPI wheel | 3 個 | files.pythonhosted.org (note v2.2.7) |
| Portal source | 1 個目錄 | git pull 你 PC 的 cl_ftp/portal/ |
| **install_portal_all_in_one.sh** | 1 個 | **本次新增** |

new sh URL:
```
https://github.com/alienid4/cl_ftp/raw/main/release-zip/install-portal-all-in-one-v2.3.2.sh
```

### 2. WinSCP 全部丟到 SF /tmp/ftp-lab/ (任一子目錄都行, 腳本自動搜尋)

可以是這樣 (混在一起也 OK):
```
/tmp/ftp-lab/
├── install-portal-all-in-one-v2.3.2.sh
├── rpms/
│   ├── python3-flask-2.0.3-1.el9.1.noarch.rpm
│   ├── python3-werkzeug-2.0.3-3.el9.1.noarch.rpm
│   ├── python3-gunicorn-21.2.0-2.el9.noarch.rpm
│   ├── python3-itsdangerous-2.0.1-4.el9.noarch.rpm
│   ├── python3-click-8.0.3-2.el9.noarch.rpm
│   ├── python3-blinker-1.5-1.el9.noarch.rpm
│   └── python3-ldap3-2.8.1-4.el9.noarch.rpm
├── Flask_Login-0.6.3-py3-none-any.whl
├── cachelib-0.10.2-py3-none-any.whl
├── flask_session-0.5.0-py3-none-any.whl
└── portal/             ← 整個目錄
    ├── wsgi.py
    └── app/
```

或全部平鋪在 `/tmp/ftp-lab/` 不分子目錄也行 (腳本自動找).

### 3. SF 跑一行

```
sudo bash /tmp/ftp-lab/install-portal-all-in-one-v2.3.2.sh
```

- 手動 debug 命令
