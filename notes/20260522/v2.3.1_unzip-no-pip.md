# Note 2026-05-22 v2.3.1 — 用 unzip 解 wheel, 不用 pip

## 情境

USER 明確規定: **公司 SF 不能用 pip**

之前 v2.2.7 / v2.2.9 用了 `pip3 install` — 違反規範, 要還原。

## 關鍵知識

**Python `.whl` 檔就是 zip 檔**, 內部結構直接對應 site-packages:

```
Flask_Login-0.6.3-py3-none-any.whl  (其實是 zip)
├── flask_login/                     ← 直接放到 site-packages 就能 import
│   ├── __init__.py
│   ├── ...
└── Flask_Login-0.6.3.dist-info/     ← metadata (pip uninstall 用, 不影響 import)
```

所以直接 `unzip` 到 `/usr/lib/python3.9/site-packages/` 等同 pip install, 但不用 pip。

## 步驟 (依序在 SF 跑)

### 0. (前置) 清掉之前 pip 留下的污染

如果 v2.2.7 / v2.2.9 跑過 pip install, 先清乾淨:

```
rm -rf /usr/local/lib/python3.9/site-packages/flask_login*
rm -rf /usr/local/lib/python3.9/site-packages/flask_session*
rm -rf /usr/local/lib/python3.9/site-packages/cachelib*
rm -rf /usr/local/lib/python3.9/site-packages/Flask_Login*
rm -rf /usr/local/lib/python3.9/site-packages/flask_session*.dist-info
rm -rf /usr/local/lib/python3.9/site-packages/cachelib*.dist-info
rm -rf /usr/lib/python3.9/site-packages/flask_login*
rm -rf /usr/lib/python3.9/site-packages/flask_session*
rm -rf /usr/lib/python3.9/site-packages/cachelib*
```

### 1. 確認 unzip 在不在 (RHEL 9 一般有)

```
which unzip || dnf install -y unzip
```

### 2. unzip 3 個 wheel 到 /usr/lib/python3.9/site-packages/

```
cd /tmp/ftp-lab

unzip -o Flask_Login-0.6.3-py3-none-any.whl -d /usr/lib/python3.9/site-packages/
unzip -o cachelib-0.10.2-py3-none-any.whl -d /usr/lib/python3.9/site-packages/
unzip -o flask_session-0.5.0-py3-none-any.whl -d /usr/lib/python3.9/site-packages/

ls -la /usr/lib/python3.9/site-packages/ | grep -E "(flask_login|flask_session|cachelib)"
```

預期看到 6 個 entries:
```
drwxr-xr-x ... cachelib
drwxr-xr-x ... cachelib-0.10.2.dist-info
drwxr-xr-x ... Flask_Login-0.6.3.dist-info
drwxr-xr-x ... flask_login
drwxr-xr-x ... flask_session
drwxr-xr-x ... flask_session-0.5.0.dist-info
```

### 3. nginx user 驗 import

```
sudo -u nginx python3 -c "from flask_login import LoginManager; from flask_session import Session; import cachelib; print('OK')"
```

預期: `OK`

### 4. 重啟 sf-portal + 驗

```
systemctl restart sf-portal
sleep 5
systemctl status sf-portal --no-pager | head -15
ss -tlnp | grep 5000
curl -s -o /dev/null -w "HTTP: %{http_code}\n" http://localhost:5000/
```

預期:
- `Active: active (running)`
- LISTEN 127.0.0.1:5000
- HTTP 200/302/401/403

### 5. PC 瀏覽器訪問

```
http://10.92.198.14/
http://10.92.198.14:5000/
```

任一個有畫面 = 成功。

---

## 為什麼這次會成功

| 之前嘗試 | 失敗原因 |
|---|---|
| `pip3 install --no-index` (v2.2.7) | 裝到 /usr/local/, nginx user sys.path 沒這條 |
| `chmod -R o+rX /usr/local/...` (v2.2.8) | 沒解決 sys.path 問題 |
| `pip3 install --prefix=/usr` (v2.2.9) | pip 可能無視 --prefix |
| **`unzip -d /usr/lib/python3.9/site-packages/`** (v2.3.1) | **直接寫入 sys.path 預設路徑, 不用 pip** |

---

## 如果 Step 3 還是 ModuleNotFoundError

代表 nginx 的 sys.path **連 /usr/lib/python3.9/site-packages/ 都沒有** (不正常的 RHEL 安裝)。看 sys.path:

```
sudo -u nginx python3 -c "import sys; print('\n'.join(sys.path))"
```

把結果貼 Claude。

## 如果 Step 4 sf-portal 起不來

代表又是新錯誤 (不是 flask_login), 跑手動 gunicorn:

```
cd /opt/portal/app
sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 --log-level debug wsgi:app 2>&1 | tail -30
```

最後 30 行貼 Claude。
