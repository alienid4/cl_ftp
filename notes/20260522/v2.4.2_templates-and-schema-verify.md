# Note 2026-05-22 v2.4.2 — 補 templates + 強化 schema 驗證 + 修 nginx 衝突

## USER 跑 v2.4.1 後狀況

跑完 install-portal-all-in-one-v2.4.1.sh 看似成功 (Active: running, gunicorn 5000 LISTEN),
但瀏覽器開 `http://10.92.198.14/auth/login?next=%2F` 顯示 **502 Bad Gateway**.

跑 v2.4.1_diag-502.sh 收集 15 段資訊, 真正 root cause 抓到 3 個:

## Root Cause 分析

### RC1: portal/app/templates/ 只有 base.html, 缺其他全部
```
jinja2.exceptions.TemplateNotFound: error.html
```
所有 blueprint route 都會 TemplateNotFound:
- login.html — auth.py
- home.html — main.py
- approval_list.html / approval_detail.html — approval.py
- audit_query.html — audit.py
- admin_biz_list.html / admin_biz_new.html / admin_biz_edit.html / admin_health.html — admin.py
- error.html — error handler (本來想接 404/500, 結果自己也炸)

→ /auth/login GET → render login.html → TemplateNotFound → 500 → render error.html → TemplateNotFound 又炸 → worker abort → nginx 收到 prematurely closed → 502.

### RC2: PG schema 沒套到 DB
```
=== 13. DB 連線測試 ===
ERROR:  relation "businesscode" does not exist
```
v2.4.1 Step 6b 只 warn 沒 fail. 萬一 schema SQL 找不到, 或 SQL 跑失敗, 表沒建 → caller 全炸.

### RC3: nginx default_server 衝突
```
nginx: [warn] conflicting server name "_" on 0.0.0.0:80, ignored
nginx: [warn] conflicting server name "_" on [::]:80, ignored
```
`/etc/nginx/nginx.conf` 內有預設 `server { listen 80 default_server; server_name _; }`,
跟 `/etc/nginx/conf.d/sf-portal.conf` 內 `server_name _` 撞名. nginx 只用其中一個.

可能進到預設 server (回 default index 或 404), 不會走到 portal proxy_pass.

## v2.4.2 修法

### 1. 補 10 個 Jinja2 templates
新建在 `portal/app/templates/`:
- error.html — 404/500 統一錯誤頁
- login.html — AD 登入表單
- home.html — 首頁 dashboard
- approval_list.html — 我的待簽列表
- approval_detail.html — 簽核細節
- audit_query.html — 稽核查詢
- admin_biz_list.html — 業務代號管理
- admin_biz_new.html — 新增業務代號
- admin_biz_edit.html — 編輯業務代號
- admin_health.html — 系統健康

設計: 全部 `{% extends "base.html" %}`, 不用 cathay.css 也能顯示基本資訊. USER 之後可換 mockup 樣式.

### 2. Step 5c 新增 templates 完整性驗證
腳本內定義 11 個必須 template (base.html + 10 個新建), 缺一個就 fail.

### 3. Step 6b 強化 schema 驗證
- 找不到 SQL → fail (不再 warn)
- 套用後驗 6 個關鍵表都在, 缺一個就 fail
- 驗證後跑 portal user 連 DB 讀 businesscode 確認權限

### 4. Step 8 修 nginx default_server 衝突
用 Python 處理 nginx.conf, 把內含 `server_name _;` 的 server { } block 整段註解.

理由: sed 處理 multi-line block 容易爆 (找不到結束括號), 用 Python 配對 brace 比較穩.

### 5. 檔名規則對齊 SKILL 鐵律 9.2
v2.4.2 起檔名一律 `v<X.Y.Z>_<name>.<ext>`:
- `v2.4.2_install-portal-all-in-one.sh` (不再 `install-portal-...-v2.4.2.sh`)
- `v2.4.2_sf-portal-source.tar.gz`
