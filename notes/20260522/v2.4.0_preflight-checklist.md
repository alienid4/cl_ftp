## ❌ v2.4.0 新發現 (還沒修)

### 7. config.py 走 dotenv 不走 appsettings.json
之前以為 `appsettings.json` 是設定檔, 結果 `config.py` 是 `from dotenv import load_dotenv`:
```python
BASE_DIR = Path(__file__).resolve().parent.parent  # = /opt/portal/app/
load_dotenv(BASE_DIR / '.env')                     # 找 /opt/portal/app/.env
```
**install 腳本必須寫 `.env`, 不是 (或不只是) `appsettings.json`.**

### 8. DB_CONNECTION_STRING 預設值是 MSSQL 連接字串
```python
DB_CONNECTION_STRING = os.getenv(
    'DB_CONNECTION_STRING',
    r'Server=.\SQLEXPRESS;...Trusted_Connection=yes;...'  # ← Windows ODBC 字串
)
```
**.env 必須明寫 `DB_CONNECTION_STRING=postgresql://portal:PASS@127.0.0.1:5432/file_exchange_audit`, 不然 db.py psycopg2.connect 會用這個 ODBC 字串連 (失敗)**

### 9. Windows 路徑預設值
config.py 各種 default 都是 Windows 風格:
```python
DATA_EXCHANGE_ROOT = os.getenv('DATA_EXCHANGE_ROOT', r'D:\DataExchange')
PORTAL_LOG_DIR = os.getenv('PORTAL_LOG_DIR', r'D:\_portal\logs')
PORTAL_BACKUP_DIR = os.getenv('PORTAL_BACKUP_DIR', r'D:\_portal\backups')
```
**.env 必須蓋掉**:
- `DATA_EXCHANGE_ROOT=/data/exchange`
- `PORTAL_LOG_DIR=/opt/portal/logs`
- `PORTAL_BACKUP_DIR=/opt/portal/backups`

### 10. dotenv 套件衝突 (v2.3.9 提過)
PyPI 上有兩個套件:
- ✅ `python-dotenv` (我們要的, 提供 load_dotenv)
- ❌ `dotenv` (stub, 沒 load_dotenv)
**腳本 unzip 前先 `rm -rf` 所有 dotenv* / python_dotenv* 殘留**

### 11. PostgreSQL schema 跟 caller code 不匹配 ⚠️ 嚴重
原 portal 程式碼是寫給 MSSQL 的, 表名是 CamelCase (PG 折成 lowercase no-underscore).
sql/01_create_db_postgres.sql 是另外寫的, 用 snake_case → caller 全部找不到表.

| Caller 用名 | PG 折成 | 現有 PG schema | 結果 |
|---|---|---|---|
| `BusinessCode` | `businesscode` | `business_code` | ❌ 找不到 |
| `Batch` | `batch` | `batch` | ✅ |
| `BatchFile` | `batchfile` | `batch_file` | ❌ |
| `AuditLog` | `auditlog` | `audit_log` | ❌ |
| `PortalUser` | `portaluser` | (沒有) | ❌ 缺表 |
| `SambaPathHistory` | `sambapathhistory` | (沒有) | ❌ 缺表 |
| `V_AuditLog_Detail` | `v_auditlog_detail` | (沒有) | ❌ 缺 view |

**v2.4.0 修法**: 重寫 `sql/01_create_db_postgres.sql`, 表名用 no-underscore lowercase 對齊 caller, 並補齊 PortalUser / SambaPathHistory / V_AuditLog_Detail.

### 12. SQL Server 語法 `SELECT TOP N` (audit_helper.py search_audit)
```python
sql = f"SELECT TOP {limit} * FROM V_AuditLog_Detail ..."
```
PostgreSQL 用 `LIMIT N` 在最後. **db.py `_translate_sql()` 必須補一條 TOP→LIMIT 翻譯**:
```python
top_match = re.search(r'\bSELECT\s+TOP\s+(\d+)\s+', sql, flags=re.IGNORECASE)
if top_match:
    n = top_match.group(1)
    sql = re.sub(r'\bSELECT\s+TOP\s+\d+\s+', 'SELECT ', sql, count=1, flags=re.IGNORECASE)
    sql = sql.rstrip().rstrip(';') + f' LIMIT {n}'
```

### 13. SELinux 可能擋 nginx 連 5000 / 讀 site-packages
- `setsebool -P httpd_can_network_connect on` (反代 5000 需要)
- 若 site-packages 是 cp 上來的, 可能無 httpd_sys_content_t context → chcon 或 restorecon

### 14. nginx 80 預設可能被 default_server 蓋住
- 寫 `/etc/nginx/conf.d/sf-portal.conf` 之前要把 `nginx.conf` 內 default_server listen 80 移掉

### 15. firewalld http 服務
- `firewall-cmd --permanent --add-service=http && --reload`
