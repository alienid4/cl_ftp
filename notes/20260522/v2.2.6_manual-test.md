# Note 2026-05-22 v2.2.6 — 手動跑 gunicorn debug Portal

## 情境

USER 跑 v2.2.5 / v2.2.6, gunicorn worker 一直 boot 失敗:
```
gunicorn.errors.HaltServer: <HaltServer 'Worker failed to boot.' 3>
restart counter is at 4286
```

systemd 把真正的 Python traceback 吞掉, 而且一直 restart 看不清楚。

**先停 systemd → 手動跑 gunicorn → 看完整錯誤訊息 → 修對應問題 → 才回頭啟動 systemd**。

## 6 步驟 (依序在 SF 主機跑)

### 1. 停 systemd 服務 (停止 restart 風暴)

```
systemctl stop sf-portal
systemctl disable sf-portal
systemctl status sf-portal | head -5
```

預期: `Active: inactive (dead)`

### 2. 手動測 wsgi import (看缺什麼模組)

```
cd /opt/portal/app
sudo -u nginx /usr/bin/python3 -c "import sys; sys.path.insert(0, '/opt/portal/app'); import wsgi; print('OK:', type(wsgi.app).__name__)"
```

可能輸出:
| 輸出 | 意義 |
|---|---|
| `OK: Flask` | wsgi.app 載入 OK, 換下一步測 gunicorn |
| `ModuleNotFoundError: No module named 'app'` | wsgi.py 找不到 app/ 子套件 (path 問題) |
| `ModuleNotFoundError: No module named 'flask_login'` | 缺套件 (記下來) |
| `ImportError: cannot import name 'create_app' from 'app'` | app/__init__.py 沒 export |
| 其他 traceback | 看具體錯誤 |

### 3. 手動跑 gunicorn (前景, 1 worker, 看完整 traceback)

```
cd /opt/portal/app
sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 --log-level debug wsgi:app
```

(注意: 用 5001 不擾既有 5000; --log-level debug 印更多細節)

按 Ctrl+C 中止。

預期最常見錯誤:

| 錯誤訊息 | 修法 |
|---|---|
| `psycopg2.OperationalError: FATAL: password authentication failed for user "portal"` | DB 密碼 mismatch, 重設 portal user |
| `psycopg2.OperationalError: could not connect to server: Connection refused` | postgresql 沒起 / port 5432 沒開 |
| `ModuleNotFoundError: No module named 'flask_login'` | Portal 用 flask_login (EPEL 9 沒有, 改 fallback) |
| `FileNotFoundError: appsettings.json` | working dir 不對 |
| `PermissionError: '/opt/portal/logs/...'` | logs 目錄 owner 不對 |

### 4. 修錯 (依錯誤訊息)

把 Step 3 的完整錯誤訊息截圖 / 貼回 Claude, Claude 寫 v2.2.7。

### 5. 如果手動 gunicorn 起得來 → 再啟動 systemd

```
systemctl enable sf-portal
systemctl start sf-portal
systemctl status sf-portal | head -10
ss -tlnp | grep 5000
```

### 6. (替代方案) 改用 Flask 內建 dev server 暫時跑

如果 gunicorn 一直修不好, 用 Flask 內建先讓 Portal 起來:

```
cd /opt/portal/app
sudo -u nginx /usr/bin/python3 -c "
from wsgi import app
app.run(host='127.0.0.1', port=5000, debug=False)
"
```

Ctrl+C 中止。如果這個跑得起來, 代表問題是 gunicorn 本身, 不是 Portal 程式碼。
