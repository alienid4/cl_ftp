# Note 2026-05-22 v2.2.7 — 補裝 flask_login / flask_session (PyPI wheels)

## 情境

USER 跑 v2.2.6 手動測得真正錯誤:
```
ModuleNotFoundError: No module named 'flask_login'
```

Portal `app/__init__.py` 用了:
- `from flask_login import LoginManager` ← 沒裝
- `from flask_session import Session` ← 沒裝

兩個套件在 **RHEL AppStream + EPEL 9 都沒有**, 必須走 PyPI wheel 離線安裝。

放到 `C:\Temp\pip-wheels\` 或任何資料夾:

## 步驟

### 1. PC 用瀏覽器點 3 個 URL 下載

到一個資料夾 (例如 `C:\Temp\pip-wheels\`)

### 2. WinSCP 把 3 個 .whl 放到 SF `/tmp/ftp-lab/`

### 3. SF 跑安裝 (不需要網路, `--no-index` 純本地)

```
cd /tmp/ftp-lab
pip3 install --no-index --no-deps Flask_Login-0.6.3-py3-none-any.whl
pip3 install --no-index --no-deps cachelib-0.10.2-py3-none-any.whl
pip3 install --no-index --no-deps flask_session-0.5.0-py3-none-any.whl
```

(必須照順序: cachelib 在 flask_session 之前)

### 4. 確認 import 通

```
python3 -c "from flask_login import LoginManager; from flask_session import Session; print('OK')"
```

預期: `OK`

### 5. 重啟 Portal

```
systemctl restart sf-portal
sleep 3
systemctl status sf-portal --no-pager | head -10
ss -tlnp | grep 5000
```

預期:
- `Active: active (running)`
- `LISTEN 0 ... 127.0.0.1:5000 ... users:(("gunicorn",pid=...))`

### 6. 從 PC 訪問

```
http://<SF-IP>/
```
或
```
http://<SF-IP>:5000/
```

看到 Portal 登入頁 (或 401/403/302) = 成功 ✓

## 為什麼用 PyPI wheel 而不是 source tarball?

- `.whl` 是 pre-built, 直接解壓到 site-packages
- `.tar.gz` 是 source, 需要 build (可能要 gcc / setuptools, 多麻煩)
- 純 Python 套件 (flask_login / flask_session 都是), `.whl` 跨平台 (`py3-none-any`)
