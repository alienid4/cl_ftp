把這個 Flask Portal 在 SF 主機上跑起來:
- **成功定義**: `systemctl is-active sf-portal` = active, `curl http://localhost:5000/` 回 HTTP 200/302/401/403, 反代 nginx 80 也通.

### config.py 關鍵行 (容易踩雷)
```python
from dotenv import load_dotenv               # 必須是 python-dotenv 不是 dotenv stub
BASE_DIR = Path(__file__).resolve().parent.parent   # = /opt/portal/app/
load_dotenv(BASE_DIR / '.env')               # 讀 /opt/portal/app/.env

# DB_CONNECTION_STRING 預設值是 MSSQL ODBC 字串 (歷史遺留), .env 必須蓋掉成 PG URL
DB_CONNECTION_STRING = os.getenv('DB_CONNECTION_STRING',
    r'Server=.\SQLEXPRESS;Database=FileExchangeAudit;Trusted_Connection=yes')
```

### db.py 關鍵 (v2.4.0)
```python
import psycopg2   # 不是 pyodbc
def _translate_sql(sql):
    sql = re.sub(r'\?', '%s', sql)                              # MSSQL ? → PG %s
    sql = re.sub(r'\bSYSUTCDATETIME\(\)', "(NOW() AT TIME ZONE 'UTC')", sql, flags=re.IGNORECASE)
    # v2.4.0 新加: SELECT TOP N → SELECT ... LIMIT N
    top_match = re.search(r'\bSELECT\s+TOP\s+(\d+)\s+', sql, flags=re.IGNORECASE)
    if top_match:
        n = top_match.group(1)
        sql = re.sub(r'\bSELECT\s+TOP\s+\d+\s+', 'SELECT ', sql, count=1, flags=re.IGNORECASE)
        sql = sql.rstrip().rstrip(';') + f' LIMIT {n}'
    return sql
```

## PostgreSQL Schema (v2.4.0)

**關鍵**: caller code 寫的是 MSSQL CamelCase 表名 (`SELECT * FROM BusinessCode`).
PostgreSQL 把 unquoted identifier 折成 lowercase, 所以 schema 必須用 **no-underscore lowercase**:

| caller 寫 | PG 折成 | schema 必須叫 |
|---|---|---|
| AuditLog | auditlog | auditlog |
| BusinessCode | businesscode | businesscode |
| Batch | batch | batch |
| BatchFile | batchfile | batchfile |
| PortalUser | portaluser | portaluser |
| SambaPathHistory | sambapathhistory | sambapathhistory |
| V_AuditLog_Detail | v_auditlog_detail | v_auditlog_detail (view) |

**完整 schema**: https://github.com/alienid4/cl_ftp/blob/main/release-zip/20260522/01_create_db_postgres_v2.4.0.sql

## 如果 v2.4.0 失敗 — Debug SOP

### Step 1: 看腳本最後印的訊息 (USER 截圖貼給你)

腳本失敗會自動印:
```
=== systemctl status sf-portal ===
=== journalctl -u sf-portal 最後 30 行 ===
=== /opt/portal/logs/error.log 最後 40 行 ===
→ 手動 debug 跑: sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 --log-level debug --chdir /opt/portal/app wsgi:app
```

### Step 2: 找第一個 Python Traceback

從上述 3 段裡找:
- `Traceback (most recent call last):`
- 最後一行 `XxxError: ...`

### Step 3: 對照常見錯 → 對應修法

| 錯訊 | 原因 | 修法 (給 USER 一行) |
|---|---|---|
| `ModuleNotFoundError: No module named 'XXX'` | XXX wheel 沒裝到 | 找對應 wheel + unzip 到 /usr/lib/python3.9/site-packages |
| `ImportError: cannot import name 'load_dotenv' from 'dotenv'` | 裝到 dotenv stub | `rm -rf /usr/lib/python3.9/site-packages/dotenv*` 再 unzip python_dotenv wheel |
| `psycopg2.OperationalError: connection ... password authentication failed` | DB 密碼跟 .env 不一致 | 重設 portal user 密碼: `sudo -u postgres psql -c "ALTER USER portal WITH PASSWORD 'xxx';"` 然後改 /opt/portal/app/.env DB_CONNECTION_STRING |
| `psycopg2.OperationalError: database "file_exchange_audit" does not exist` | DB 沒建 | `sudo -u postgres psql -c "CREATE DATABASE file_exchange_audit;"` |
| `psycopg2.errors.UndefinedTable: relation "xxx" does not exist` | schema 沒套用 | `sudo -u postgres psql -d file_exchange_audit -f /tmp/sf-portal-extracted/portal/sql/01_create_db_postgres.sql` |
| `KeyError: 'DB_CONNECTION_STRING'` | .env 沒讀到 | 檢查 `/opt/portal/app/.env` 存在且 RUN_USER (nginx) 可讀 |
| `permission denied: /usr/lib/python3.9/site-packages/xxx` | site-packages 權限不對 | `chmod -R o+rX /usr/lib/python3.9/site-packages/` |
| `[error] connect() to 127.0.0.1:5000 failed (13: Permission denied)` (nginx log) | SELinux 擋 | `setsebool -P httpd_can_network_connect on` |
| `Worker failed to boot` (沒具體錯) | gunicorn 把錯吃掉 | 跑手動 gunicorn 看完整 traceback (Step 4) |

### Step 4: 手動 gunicorn 撈完整 traceback

```bash
cd /opt/portal/app
sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 \
    --log-level debug --chdir /opt/portal/app wsgi:app
```

按 Ctrl+C 中止. 完整 traceback 會印 stdout.

### Step 5: 如果你 (GPT) 確定要改檔, 不要叫 USER 複製貼上

- 寫一個 `.sh` 腳本貼到 GitHub raw URL, 叫 USER 下載 + 執行
- 或寫一條 USER 可以手打的單行指令 (< 80 字元)
- 不要寫多行指令給 USER 複製貼上 (DLP 會擋)

## 不要做的事 (給接手 AI)

1. ❌ 不要叫 USER `pip install` — 公司禁
2. ❌ 不要叫 USER `dnf install epel-release` — 公司禁 (用 Satellite)
3. ❌ 不要叫 USER 從 internet `curl/wget` 下載 — SF 沒 internet
4. ❌ 不要假設 `/usr/local/lib/python3.9/site-packages` 在 sys.path — nginx user 看不到
5. ❌ 不要假設 RHEL 9 有 `gunicorn-3` binary — 是 `/usr/bin/gunicorn`
6. ❌ 不要寫 MSSQL syntax 給 PG (TOP N, SCOPE_IDENTITY, SYSUTCDATETIME 等)
7. ❌ 不要在 PostgreSQL DO block 內 CREATE DATABASE
8. ❌ 不要在 PG schema 用 snake_case (caller 是 CamelCase 折成 no-underscore lowercase)
9. ❌ 不要寫多行指令給 USER 複製 — DLP 擋
10. ❌ 不要 `git push --force` 任何分支
11. ❌ 不要假設可以連 dl.fedoraproject.org / pypi.org
