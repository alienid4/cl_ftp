## 3 個指令找真正錯誤 (依序跑)

### 1. 看 gunicorn 自己的 error log (不是 systemd 的)

systemd 寫 `portal-stderr.log` 是包裝外層, 真正 Python error 在 gunicorn `--error-logfile` 寫的 `error.log`:

```
cat /opt/portal/logs/error.log
```

或:

```
head -100 /opt/portal/logs/error.log
```

裡面第一個 `Traceback` 起頭那段就是真正錯誤 (例如 `ModuleNotFoundError: XXX`, `ImportError: ...`, `psycopg2.OperationalError: ...`)。

### 2. 看 stderr.log 完整 (從頭看, 不要 tail)

```
head -100 /opt/portal/logs/portal-stderr.log
```

第一個 Traceback 才是真錯誤, 後面 HaltServer 是 gunicorn 自己 wrap 的二次錯誤。

### 3. 手動跑 gunicorn (用 --chdir 強制 working dir, 看完整輸出)

```
sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 --log-level debug --chdir /opt/portal/app wsgi:app
```

(加 `--chdir /opt/portal/app` 確保 working dir 對, 不需要先 cd)

按 Ctrl+C 中止。

## 預期看到 (最可能錯誤類型)

| 錯誤 | 修法 |
|---|---|
| `ModuleNotFoundError: No module named 'XXX'` | 又缺 PyPI 套件, 補 wheel |
| `psycopg2.OperationalError: FATAL: password authentication failed for user "portal"` | DB 密碼 mismatch, 重設 portal user |
| `psycopg2.OperationalError: could not translate host name` | DATABASE_URL 寫錯 |
| `FileNotFoundError: '/data/exchange'` | DATA_ROOT 缺 (v2.3.4 應該建了, 確認 `ls -la /data/`) |
| `PermissionError: '/opt/portal/logs/...'` | logs 目錄 owner 不對 |
| `AttributeError: module 'flask_login' has no attribute 'X'` | flask_login 版本不相容 |
| `RuntimeError: SECRET_KEY` | appsettings.json 沒寫好 SECRET_KEY |
