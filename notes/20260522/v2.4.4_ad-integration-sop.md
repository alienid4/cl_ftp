# Note 2026-05-22 v2.4.4 — SF Portal × AD 整合 SOP

## 概觀

SF Portal 用 AD 認證 + AD 群組授權, **不存個人密碼**.

三個角色:
1. **個人 AD 帳號** (CORP\xxx) — USER 登入用, Portal 直接對 AD bind 驗證
2. **服務帳號** (svc_portal_ldap) — Portal 用來查 AD 群組成員, 不存個人密碼
3. **AD 群組** (g_u0X_approvers / sf_admin / dept_xxx_view) — 控制誰能簽 / 誰能下載 / 誰是 IT

---

## A. AD 管理員要設定的 (一次性, 上線前)

### A1. 建服務帳號

```
名稱: svc_portal_ldap
類型: User Account (非 Computer)
密碼策略: 設長密碼 (16+ 字元), 永不過期 (或定期 rotate 通知 IT)
用途: SF Portal 查 AD 群組成員
```

權限要求:
- 一般 Domain Users 即可 (能讀自己 OU)
- 不需要 Admin / Account Operators
- **僅需 Read 權限**, 不能改 AD

把密碼放 PAM 或公司密碼金庫. 不存在程式碼.

### A2. 建簽核 AD 群組 (依業務代號)

| 群組名 | OU 建議 | 用途 | 成員 |
|---|---|---|---|
| `g_u01_approvers` | `OU=SF,OU=Groups,DC=corp,DC=local` | u01 業務簽核人 (5 人 ANY 制) | 該業務的 5 位簽核人 (個人帳號) |
| `g_u02_approvers` | 同上 | u02 業務簽核人 | 5 位 |
| `g_u03_approvers` | 同上 | u03 業務簽核人 | 5 位 |
| `g_u04_approvers` | 同上 | u04 業務簽核人 | 5 位 |

群組類型: **Universal** 或 **Global Security Group** (依公司 forest 結構)

維護: 之後 Portal 內「我的群組」頁讓**業務負責人自助維護**, IT 不介入個別群組成員調動. (前提: IT 給業務負責人對該群組的「Modify Members」delegation)

### A3. 建下載 AD 群組 (依部門 SMB 路徑)

| 群組名 | 用途 |
|---|---|
| `dept_archi_view` | 架構科可下載 (對應 samba\architecture\) |
| `dept_hr_view` | HR 可下載 |
| `dept_finance_view` | 財務可下載 |
| `dept_security_view` | 資安可下載 |

### A4. 建 IT 管理員群組

```
名稱: sf_admin
成員: IT 維運該系統的 2~3 人
用途: Portal admin 介面 (業務代號管理 / 稽核 / 系統健康)
```

Portal 認任何 group name 含 `sf_admin` 或 `it.admin` 的人為 admin (見 auth.py:132).

### A5. 給 Delegation (業務負責人自助維護用)

對每個 `g_u0X_approvers` 群組, 設:
- **Managed By** = 該業務負責人個人帳號
- 勾「Manager can update membership list」

USER 端: 開 Portal「我的群組」頁 (前提是該頁有實作), 可加減成員. **不用 IT 介入**.

或: AD 管理員開「Active Directory Users and Computers」, 右鍵群組 → Delegate Control → 給 owner 「Read membership」+「Write members」.

### A6. 防火牆放行

從 SF 主機 (10.92.198.14) 對 DC server:
- TCP 389 (LDAP, 明文)
- TCP 636 (LDAPS, 加密, **建議用這個**)
- TCP 3268 (Global Catalog)
- TCP 3269 (Global Catalog Secure)

## B. SF 端要改的 (.env 設定)

### B1. 取 AD 資訊 (跟 AD 管理員要)

問 AD 管理員拿到:
- DC server IP 或 hostname (例: `10.10.0.1` 或 `dc01.cathay.com`)
- Domain NetBIOS 名 (例: `CATHAY`, `CORP`, `INTRA`)
- Base DN (例: `DC=cathay,DC=com` 或 `DC=corp,DC=local`)
- 服務帳號 username 與密碼

### B2. 改 /opt/portal/app/.env

關鍵 5 行 (其他保留):

```env
# DEV_MODE 上線前必須改 false
DEV_MODE=false

# AD bind 設定
AD_SERVER=ldap://dc01.cathay.com:389
# 或用 LDAPS (推薦, 加密):
# AD_SERVER=ldaps://dc01.cathay.com:636

AD_BASE_DN=DC=cathay,DC=com
AD_DOMAIN=CATHAY

AD_BIND_USER=CATHAY\svc_portal_ldap
AD_BIND_PASS=<服務帳號真實密碼, 從 PAM 拿>
```

注意:
- `AD_BIND_USER` 要含 domain prefix (`CATHAY\svc_portal_ldap`), 不是只 `svc_portal_ldap`
- `AD_BIND_PASS` 不寫死, 之後改用 PAM API 取
- `AD_SERVER` 多個 DC 可以寫: `AD_SERVER=ldap://dc01,dc02,dc03.cathay.com` (ldap3 支援 round-robin)

### B3. 重啟 sf-portal

```
sudo systemctl restart sf-portal
sleep 3
sudo systemctl status sf-portal --no-pager
```

## C. 驗證 AD bind 通了

### C1. 從 SF terminal 手動測 ldap

```
sudo dnf install openldap-clients -y
ldapsearch -x -H ldap://dc01.cathay.com \
    -D "CATHAY\svc_portal_ldap" -w '<密碼>' \
    -b "DC=cathay,DC=com" \
    "(sAMAccountName=01003385)" \
    cn mail memberOf
```

預期: 列出該 USER 的 cn, mail, memberOf 清單.

若回 `Invalid credentials` → 帳密不對 / domain 錯
若回 `Can't contact LDAP server` → 防火牆或 DNS 問題

### C2. Python 端測

```
sudo -u nginx /usr/bin/python3 <<'PY'
from ldap3 import Server, Connection, ALL, NTLM
s = Server('ldap://dc01.cathay.com', get_info=ALL)
c = Connection(s, user='CATHAY\\svc_portal_ldap', password='<密碼>',
               authentication=NTLM, auto_bind=True)
c.search('DC=cathay,DC=com', '(sAMAccountName=01003385)', attributes=['memberOf'])
print(c.entries[0])
c.unbind()
PY
```

### C3. Portal 端測

瀏覽器開 `http://10.92.198.14/`, login 頁輸入:
- AD 帳號 = 你的個人 AD (例 `CATHAY\01003385`)
- 密碼 = 你的個人 AD 密碼

預期:
- 成功 → 進 home.html, 上方顯示你的 displayName
- 失敗 → flash 訊息「AD bind 失敗」, 看 portal-stderr.log 找原因

## D. 常見錯誤對照

| 錯訊 | 意義 | 修法 |
|---|---|---|
| `Can't contact LDAP server` | DC 沒通 | 檢查防火牆 + DNS + `AD_SERVER` URL |
| `Invalid credentials` | AD_BIND_USER / PASS 不對 | 重設服務帳號密碼, 同步 .env |
| `LDAPStrongerAuthRequiredResult` | 公司禁明文 LDAP | 改用 LDAPS (port 636) |
| `LDAPInvalidDnSyntaxResult` | AD_BASE_DN 寫錯 | 應該是 `DC=cathay,DC=com` 不是 `cathay.com` |
| `entries 為空` | sAMAccountName 找不到 | OU 範圍不對, AD_BASE_DN 太小 |
| Portal 進去但 USER 不是 admin | 群組沒含 `sf_admin` | 加 USER 到 sf_admin 群組, 重新登入 |
| 簽核頁看不到批次 | 群組沒含 `g_u0X_approvers` | 業務負責人在 Portal「我的群組」加你 |

## E. LDAPS (建議啟用) 額外步驟

### E1. AD 管理員端

DC 主機要安裝 CA 簽的憑證 (或公司內部 CA):
```
DC: Server Manager → Add Roles → Active Directory Certificate Services
產生憑證, 綁 DC FQDN, 開啟 LDAPS 服務 (port 636 自動聽)
```

### E2. SF Portal 端

把公司 CA 公鑰加進信任清單:
```
sudo cp company-ca.crt /etc/pki/ca-trust/source/anchors/
sudo update-ca-trust
```

.env 改:
```
AD_SERVER=ldaps://dc01.cathay.com:636
```

驗:
```
openssl s_client -connect dc01.cathay.com:636 -showcerts
```
看到 `Verify return code: 0 (ok)` = 信任鏈 OK.

## F. PAM 整合 (階段二, 之後做)

目前: AD_BIND_PASS 寫在 .env (明文)
階段二: 從公司 PAM 取密碼 + auto rotate

實作: 寫個 helper `pam_get_password.py`, .env 改成:
```
AD_BIND_PASS_FROM_PAM=svc_portal_ldap
```

config.py 偵測到這個變數, 啟動時去 PAM API 抓密碼 (Bearer token 或 mTLS).
