# Note 2026-05-22 v2.3.9 — dotenv 套件衝突, 抓出來

## 情境

新錯誤訊息變了:
```
ImportError: cannot import name 'load_dotenv' from 'dotenv' (unknown location)
```

意義: dotenv 套件**存在** (能 import), 但裡面**沒 load_dotenv 函式**。

代表系統上裝了**錯的 dotenv 套件**:
- ❌ `dotenv` (PyPI 上獨立的 stub 套件, 無 load_dotenv)
- ✅ `python-dotenv` (這個才提供 load_dotenv) — 我們想要的

PyPI 有兩個套件名類似:
| 套件名 | 安裝後 import 名 | 有 load_dotenv? |
|---|---|---|
| python-dotenv | `dotenv` | ✅ 有 |
| dotenv (其他人發的) | `dotenv` | ❌ 沒 (是 stub) |

兩個共用同名 import target → 後裝會蓋掉前裝, 或被 namespace conflict 蓋住。

## 3 步診斷 + 清理

### Step 1: 找系統上所有 dotenv 安裝位置

```
find / -name 'dotenv*' -type d 2>/dev/null
find / -name 'dotenv*' -type f -name '*.py' 2>/dev/null | head -20
find / -name '*dotenv*.dist-info' -type d 2>/dev/null
find / -name '*dotenv*.egg-info' -type d 2>/dev/null
```

預期可能看到:
- `/usr/lib/python3.9/site-packages/dotenv` (target, 應該存在)
- `/usr/lib/python3.9/site-packages/python_dotenv-1.0.1.dist-info` (target)
- `/usr/local/lib/python3.9/site-packages/dotenv` (污染源? 可能是錯套件)
- `/usr/lib/python3.9/site-packages/dotenv.py` (stub, 蓋掉真套件)

### Step 2: 看當前載到哪個 dotenv

```
python3 -c "import dotenv; print(dotenv.__file__); print(dir(dotenv))"
```

正確輸出應該:
```
/usr/lib/python3.9/site-packages/dotenv/__init__.py
['load_dotenv', 'dotenv_values', 'find_dotenv', 'get_key', 'set_key', 'unset_key', ...]
```

錯誤輸出 (現況):
```
/usr/lib/python3.9/site-packages/dotenv.py    ← 是檔不是目錄, 是 stub
[]    ← 空的, 沒 load_dotenv
```

### Step 3: 清掉污染 + 重 unzip

```
# 清所有 dotenv 痕跡
rm -rf /usr/lib/python3.9/site-packages/dotenv* /usr/lib/python3.9/site-packages/python_dotenv*
rm -rf /usr/local/lib/python3.9/site-packages/dotenv* /usr/local/lib/python3.9/site-packages/python_dotenv*

# 重新 unzip python_dotenv wheel
cd /tmp/ftp-lab
ls python_dotenv*.whl    # 確認檔在
unzip -o python_dotenv-1.0.1-py3-none-any.whl -d /usr/lib/python3.9/site-packages/

# 驗證裝對了
python3 -c "import dotenv; print(dotenv.__file__); print(hasattr(dotenv, 'load_dotenv'))"
```

預期:
```
/usr/lib/python3.9/site-packages/dotenv/__init__.py
True
```

### Step 4: nginx user 也驗

```
sudo -u nginx python3 -c "from dotenv import load_dotenv; print('OK')"
```

預期: `OK`

### Step 5: 重啟 sf-portal

```
systemctl restart sf-portal
sleep 5
systemctl status sf-portal --no-pager | head -10
ss -tlnp | grep 5000
curl -s -o /dev/null -w "HTTP: %{http_code}\n" http://localhost:5000/
```

預期:
- `Active: active (running)`
- LISTEN 127.0.0.1:5000
- HTTP 200/302/401/403
