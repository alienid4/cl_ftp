# Note 2026-05-22 v2.4.1 — Python 3.9 不支援 `X | None` typing 語法

## 背景

USER 跑 v2.4.0 安裝腳本後仍失敗, 公司 GPT 接手 debug, 最終找出 root cause:

**Portal 程式碼用 PEP 604 union syntax (`User | None`), RHEL 9 系統 Python 3.9 不支援 (要 Python 3.10+)**

## 致命錯訊

```
/opt/portal/logs/error.log:
def load_user(ad_account: str) -> User | None:
TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'
```

import 階段直接炸, gunicorn worker boot 失敗.

## v2.4.0 預檢漏掉的 — 反省

pre-flight 那次掃了所有 import / SQL / 設定路徑, 但**沒掃 typing annotation 語法**.
規則:

> **Py3.9 不支援的語法**:
> - `X | Y` (PEP 604 union, 3.10+)
> - `from typing import Self` (3.11+)
> - match-case (3.10+)
> - `*` in `def f(*, x: T)` 沒問題 (3.0+)
>
> **Py3.9 支援的** (PEP 585):
> - `list[int]` (3.9 OK)
> - `dict[str, int]` (3.9 OK)
> - `tuple[X, ...]` (3.9 OK)

## 範圍

`grep -rE ' \| ' portal/app/*.py` 結果只 2 行 (都在 auth.py):

```
portal/app/auth.py:34: def load_user(ad_account: str) -> User | None:
portal/app/auth.py:55: def authenticate(username: str, password: str) -> User | None:
```

`list[str]` / `list[dict]` 各 1 處 (auth.py:142, audit_helper.py:112) — **不用改**, PEP 585 在 Py3.9.

## v2.4.1 修法

### portal/app/auth.py
```python
"""
SF Portal — AD/LDAP 認證 + User Model
"""
from typing import Optional      # ← v2.4.1 新加
from flask_login import UserMixin
...

def load_user(ad_account: str) -> Optional[User]:    # 之前: -> User | None:
    ...

def authenticate(username: str, password: str) -> Optional[User]:    # 之前: -> User | None:
    ...
```

### 公司 GPT 第一次踩雷 (供未來注意)

GPT 第一版 sed 把 `from typing import Optional` 插進了 docstring 三引號內:

```python
"""
from typing import Optional   ← 還在 docstring 內, 沒真的 import
SF Portal — ...
"""
```

結果: `NameError: name 'Optional' is not defined`

**教訓**: 修檔頭 import 不要用無腦 sed, 用 ast 解析或人眼確認 docstring 範圍.

v2.4.1 install 腳本內的 fallback patch 用 Python 解析, 不用 sed.

## 教訓 (給 pre-flight 規則加一條)

**鐵律 12.2**: pre-flight 掃語法時, 把目標 Python 版本當 baseline 跑一遍 ast/compile:

```bash
# v2.4.1 起 pre-flight 加這條
docker run --rm -v $PWD/portal:/p python:3.9 \
    python -c "import py_compile, pathlib; \
               [py_compile.compile(str(f), doraise=True) for f in pathlib.Path('/p').rglob('*.py')]"
```

(SF 沒 docker, 但 PC 有 Py3.14 → 不能用. 換用本機任一 Python 跑 `ast.parse()` 不能抓出 `|` 因為 ast 不 evaluate. 唯一辦法: 真的用 Py3.9 跑 import.)

替代方案: pre-flight grep 黑名單:
```bash
grep -rnE ' \| (None|[A-Z][a-zA-Z]+)' portal/*.py
grep -rn 'match\s.*:$' portal/*.py
```
