# Note 2026-05-22 v2.4.4 — 加 DEV_MODE 跳過 AD bind

## 情境

USER 跑 v2.4.3 起來看到 login.html ✅, 但問「帳號密碼是什麼?」

Portal 走 AD bind, 沒預設帳密. .env 內 AD_SERVER 還是 placeholder `corp-dc01.corp.local`, USER 沒設好公司 AD, 試任何帳密都會 fail.

USER 選最快路: 加 dev mode bypass.

## 修法

### portal/app/auth.py 兩處加 DEV_MODE bypass

**load_user()** (session 還原 user):
```python
def load_user(ad_account):
    import os
    if os.getenv('DEV_MODE', '').lower() in ('1', 'true', 'yes'):
        return User(ad_account=ad_account, ..., is_admin=True, groups=['sf_admin', ...])
    # 原 DB 查邏輯
```

**authenticate()** (登入):
```python
def authenticate(username, password):
    if os.getenv('DEV_MODE', '').lower() in ('1', 'true', 'yes'):
        return User(ad_account=username or 'devadmin', ..., is_admin=True, groups=[...])
    # 原 AD bind 邏輯
```

DEV_MODE 預設給 admin 身分 + 在 g_u01~g_u04 所有 approver 群組內. 任何頁面都進得去.

### .env 加 DEV_MODE=true

config.py 不用改 — `os.getenv('DEV_MODE')` 由 dotenv 自動 load 進 process env.

## 上線前必關

```
sudo sed -i 's|^DEV_MODE=.*|DEV_MODE=false|' /opt/portal/app/.env
sudo systemctl restart sf-portal
```

並且要設好:
- `AD_SERVER=ldap://<公司實際 DC IP 或 hostname>`
- `AD_DOMAIN=<公司實際 NetBIOS 名, 例 CATHAY>`
- `AD_BASE_DN=DC=<your>,DC=local`
- `AD_BIND_USER` / `AD_BIND_PASS` (svc_portal_ldap 之類服務帳號)
