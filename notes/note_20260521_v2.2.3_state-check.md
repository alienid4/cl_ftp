# Note 2026-05-21 v2.2.3 — Portal 沒設定起來的狀態檢查

## 情境

USER 跑完 `fix-portal-v2.2.3.sh` 印 DONE, 但回報:
- 「根本沒有設定 sf-portal」
- `systemctl status sf-portal` 應該回 `Unit sf-portal.service could not be found`

代表 fix_portal Step 4 (寫 sf-portal.service) **沒跑到**, 腳本中途 abort 了。

## 5 個狀態檢查 (依序跑, 全部跑完一次截圖)

### 1. EPEL Python 真的裝起來?

```
rpm -qa | grep -E "python3-(flask|werkzeug|gunicorn|itsdangerous|click|blinker|ldap3)" | sort
```

預期: 7 行 (v2.2.2 已驗證過, 應該 OK)

### 2. Portal 程式碼到 /opt/portal/app?

```
ls -la /opt/portal/app/ | head -20
```

預期: 有 `wsgi.py`, `appsettings.json` 等

### 3. appsettings.json 內容?

```
cat /opt/portal/app/appsettings.json
```

預期: JSON 含 DATABASE_URL / SECRET_KEY / PORTAL_PORT=5000

如果**檔案不存在** → Step 3 沒跑完

### 4. sf-portal.service 寫了嗎?

```
ls -la /etc/systemd/system/sf-portal.service 2>&1
```

預期: 顯示檔案

如果 `No such file` → **Step 4 沒跑到**, 腳本在 Step 3 abort 了

### 5. PostgreSQL 跑著嗎?

```
systemctl is-active postgresql && sudo -u postgres psql -c "\l" 2>&1 | head -10
```

預期: `active` + 列 database 含 `file_exchange_audit`

如果 DB 不存在 → CREATE DATABASE 失敗 → 後面全部失敗
