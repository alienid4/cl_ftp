# Note 2026-05-22 v2.3.0 — 手動 cp 套件到 /usr/lib (跳過 pip)

## 情境

USER 跑完 v2.2.9 (pip3 install --prefix=/usr) 後仍 HTTP 000 — 大概是:
- pip3 不認 `--prefix` 或裝去別地方
- 或裝對位置但仍有別的問題

**直接手段**: 跳過 pip, 把套件**手動 cp** 到 `/usr/lib/python3.9/site-packages/` (確定在 sys.path)。

## 4 步 (依序跑)

### 1. 看現在 flask_login 在哪

```
find / -name 'flask_login' -type d 2>/dev/null
find / -name 'flask_session' -type d 2>/dev/null
find / -name 'cachelib' -type d 2>/dev/null
```

預期看到 (其中之一):
- `/usr/local/lib/python3.9/site-packages/flask_login` (v2.2.7 裝的)
- `/usr/lib/python3.9/site-packages/flask_login` (v2.2.9 真的裝對了)
- 兩個都有 (v2.2.9 沒完整 uninstall)

### 2. 手動 cp 到 RHEL 標準位置

```
# 從 /usr/local/ cp 到 /usr/ (跳過 pip, 直接複製)
cp -r /usr/local/lib/python3.9/site-packages/flask_login /usr/lib/python3.9/site-packages/ 2>&1 | head -3
cp -r /usr/local/lib/python3.9/site-packages/flask_session /usr/lib/python3.9/site-packages/ 2>&1 | head -3
cp -r /usr/local/lib/python3.9/site-packages/cachelib /usr/lib/python3.9/site-packages/ 2>&1 | head -3

# .dist-info 一起 (有些 metadata 用)
cp -r /usr/local/lib/python3.9/site-packages/Flask_Login*.dist-info /usr/lib/python3.9/site-packages/ 2>/dev/null
cp -r /usr/local/lib/python3.9/site-packages/flask_session*.dist-info /usr/lib/python3.9/site-packages/ 2>/dev/null
cp -r /usr/local/lib/python3.9/site-packages/cachelib*.dist-info /usr/lib/python3.9/site-packages/ 2>/dev/null
```

如果 Step 1 看到的位置不是 `/usr/local/...`, 改成那個路徑 cp。

### 3. 確認 nginx 看得到

```
ls -la /usr/lib/python3.9/site-packages/flask_login/ | head -3
sudo -u nginx python3 -c "from flask_login import LoginManager; from flask_session import Session; print('OK')"
```

預期: `OK`

### 4. 重啟 + 驗

```
systemctl restart sf-portal
sleep 5
systemctl status sf-portal --no-pager | head -15
ss -tlnp | grep 5000
curl -s -o /dev/null -w "HTTP: %{http_code}\n" http://localhost:5000/
```

預期:
- `Active: active (running)`
- LISTEN 127.0.0.1:5000
- HTTP: 200/302/401/403

## 如果 Step 3 仍 ModuleNotFoundError

完全跳過 pip, 直接從 wheel 解壓 (wheel 就是 zip):

```
cd /tmp/ftp-lab
unzip -o Flask_Login-0.6.3-py3-none-any.whl -d /usr/lib/python3.9/site-packages/
unzip -o cachelib-0.10.2-py3-none-any.whl -d /usr/lib/python3.9/site-packages/
unzip -o flask_session-0.5.0-py3-none-any.whl -d /usr/lib/python3.9/site-packages/
sudo -u nginx python3 -c "import flask_login; print(flask_login.__file__)"
```

(`unzip` 沒裝就 `dnf install -y unzip`)

## 如果 Step 4 sf-portal 還是起不來

代表問題不是 flask_login 了 — 跑手動 gunicorn 看新 traceback:

```
cd /opt/portal/app
sudo -u nginx /usr/bin/gunicorn --workers 1 --bind 127.0.0.1:5001 --log-level debug wsgi:app 2>&1 | tail -30
```

把最後 30 行 (含 Traceback) 截圖貼 Claude。

## 對齊 sys.path 直覺

RHEL 9 系統 python3 預設 sys.path 包含:
```
/usr/lib64/python3.9
/usr/lib/python3.9
/usr/lib/python3.9/site-packages
/usr/lib64/python3.9/site-packages
```

**不包含** `/usr/local/...` (除非 root 有 PYTHONPATH / .pth 檔)

所以套件要在 `/usr/lib/python3.9/site-packages/` 才會被找到。
