# Note 2026-05-22 v2.2.8 — 修 /usr/local/lib python3 site-packages 權限

## 情境確認

USER Step 1+2 跑出來:
- root user: `flask_login` 在 `/usr/local/lib/python3.9/site-packages/flask_login/__init__.py`
- nginx user: `ModuleNotFoundError: No module named 'flask_login'`

問題: `/usr/local/lib/python3.9/site-packages/` 或其子目錄某層權限 nginx 讀不到。

## 修法 (2 個指令)

### 1. 改全部 o+rX (讓 other 也能讀)

```
chmod -R o+rX /usr/local/lib/python3.9/site-packages/
```

### 2. 驗 + 重啟

```
sudo -u nginx python3 -c "from flask_login import LoginManager; from flask_session import Session; print('OK')"
systemctl restart sf-portal
sleep 3
systemctl status sf-portal --no-pager | head -8
ss -tlnp | grep 5000
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:5000/
```

預期:
- `OK`
- `Active: active (running)`
- `LISTEN 0 ... 127.0.0.1:5000 ... users:(("gunicorn"...))`
- `200` or `302` or `401` or `403`

### 3. PC 瀏覽器訪問

```
http://10.92.198.14/
http://10.92.198.14:5000/
```

任一個有畫面 = 成功。

---

## 為什麼 chmod 修就好

- pip3 root install 時權限可能 600 (只有 owner 讀)
- `chmod -R o+rX`:
  - `o+r` 給其他 user 讀檔權
  - 大寫 `X` 只給有 execute (目錄走入) 的目錄 +x, 不會誤給檔案 +x
- 是一次性修法, 之後 pip install 應該自動有對的權限

---

## 如果 chmod 後還是不行

跑 `pip3 install --no-index --no-deps --force-reinstall --prefix=/usr` 重裝到 RHEL 標準位置 `/usr/lib/python3.9/site-packages/`:

```
cd /tmp/ftp-lab
pip3 install --no-index --no-deps --force-reinstall --prefix=/usr Flask_Login-0.6.3-py3-none-any.whl cachelib-0.10.2-py3-none-any.whl flask_session-0.5.0-py3-none-any.whl
systemctl restart sf-portal
```
